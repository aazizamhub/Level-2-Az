Model Training & Validation

Big Data Projects — Subtopic 2 of 2

⏱ 60 min total
📖 30 min theory
📝 30 min practice
⚖️ Exam weight ~6%
Learning Objectives — Hari ini lo harus bisa:
  • Bedakan fungsi training sample, validation (CV) sample, dan test sample; serta jelaskan kenapa 60:20:20 split adalah rekomendasi default
  • Jelaskan bias–variance tradeoff: kapan model underfitting vs overfitting, dan dua strategi utama untuk mitigasi (complexity reduction & cross-validation)
  • Hitung confusion matrix metrics (precision, recall, F1, accuracy) serta interpretasikan ROC/AUC dan RMSE untuk assess model performance
  • Terapkan hyperparameter tuning via grid search dan fitting curves; pahami peran ceiling analysis di complex model pipelines
Theory · 30 min
1
Method Selection & Dataset Partitioning (60:20:20)

Method selection — tiga faktor utama: (1) supervised vs unsupervised; (2) type of data (numerik, text, image, speech); (3) size of data.

Data TypeMethod yang Cocok
NumericalCART, regression, ensemble methods
TextGLMs, SVMs, Naive Bayes
ImageNeural Networks, deep learning (CNN)
SpeechDeep learning (RNN, LSTM)
"Wide" (few obs, many features)SVM
"Long" (many obs, few features)Neural Networks

Partitioning master dataset — untuk supervised learning, pecah jadi 3 subsets (tipical 60:20:20):

Training sample ≈ 60% → fit model parameters (in-sample)
Validation (CV) sample ≈ 20% → tune hyperparameters (in-sample)
Test sample ≈ 20% → assess final performance (out-of-sample)
// Test set TIDAK disentuh sampai final evaluation.
Key Insight: Training + validation = "in-sample"; test = "out-of-sample". Kalau lo ulang-ulang tune model berdasarkan test set, test set jadi "in-sample" — overfitting bocor ke out-of-sample estimate. Inilah kenapa validation set dipisah dari test set. Unsupervised learning tidak perlu splitting karena tidak ada ground truth (target variable).
⚠ Exam Trap

Class imbalance: kalau salah satu kelas dominan (misalnya default rate 2%), model naive bisa dapat accuracy 98% hanya dengan prediksi "tidak default" semua. Solusi: undersample majority class atau oversample minority class sebelum training. Ingat: F1 score lebih tepat dari accuracy untuk imbalanced data.

2
Bias–Variance Tradeoff & Cross-Validation

Total error = Bias² + Variance + Irreducible noise. Dua tipe error yang bisa dikontrol:

ErrorGejalaPenyebab
Bias (underfit)Training error tinggi, CV error tinggi (keduanya)Model terlalu simple; fitur kurang; regularization terlalu besar
Variance (overfit)Training error kecil, CV error jauh lebih besarModel terlalu complex; fitur terlalu banyak; data training sedikit

Dua strategi mitigasi overfitting:

1. Complexity reduction (Occam's razor): tambahkan overfitting penalty
— mis. L1/L2 regularization, prune decision tree
2. Proper data sampling: k-fold cross-validation
— shuffle data, bagi jadi k equal sub-samples
— k-1 untuk train, 1 untuk validate; ulang k kali
— k biasanya 5 atau 10
— E_CV ≈ rata-rata k validation errors ≈ estimate of E_out
Key Insight: k-fold CV memastikan tiap data point dipakai k-1 kali untuk training dan 1 kali untuk validation — mengurangi sampling bias tanpa mengorbankan ukuran training set. Limitasi: k-fold tidak cocok untuk time-series data (leakage: data masa depan bocor ke training). Untuk time-series, gunakan rolling-window atau walk-forward validation.
3
Performance Metrics: Confusion Matrix, ROC/AUC, RMSE

Confusion Matrix untuk binary classification — 4 sel:

Actual = 1Actual = 0
Predicted = 1TP (true positive)FP (Type I error)
Predicted = 0FN (Type II error)TN (true negative)
Precision (P) = TP / (TP + FP) // dari yang diprediksi positif, berapa % benar
Recall (R) = TP / (TP + FN) // dari semua positif aktual, berapa % terdeteksi (= TPR)
Accuracy = (TP + TN) / total
F1 score = 2·P·R / (P + R) // harmonic mean; pakai untuk imbalanced data
FPR = FP / (TN + FP)
TPR = TP / (TP + FN) = Recall
// ROC: plot TPR (y) vs FPR (x) di berbagai threshold. AUC = area under curve.
// AUC=0.5 random guess; AUC=1.0 perfect. Semakin konveks ROC, semakin baik.

RMSE = √[ Σ(predicted − actual)² / n ] // untuk regression / continuous
Kapan pakai yang mana? Precision tinggi penting ketika FP mahal (mis. scrap produk padahal bagus). Recall tinggi penting ketika FN mahal (mis. lolos produk cacat ke pelanggan, atau fraud yang lolos deteksi). F1 untuk balanced view di kelas imbalanced. AUC untuk model ranking independent of threshold. RMSE untuk regression.
⚠ Exam Trap

Accuracy MISLEADING di imbalanced dataset. Contoh: 1000 bond, 980 non-default, 20 default. Model yang prediksi semua "non-default" → accuracy 98% tapi recall = 0%! Soal CFA sering tes apakah lo bisa pilih F1 over accuracy di kasus imbalanced.

4
Hyperparameter Tuning, Fitting Curves & Ceiling Analysis

Parameters vs Hyperparameters:

ParametersHyperparameters
SourceDipelajari dari data trainingDi-set manual oleh researcher
ContohRegression coefficients, NN weights, SVM support vectorsλ (regularization), # hidden layers, # trees, k (KNN/k-means), p-threshold (logistic)

Grid Search — metode sistematis untuk cari kombinasi hyperparameter optimal. Train model dengan berbagai kombinasi, cross-validate, pilih yang paling kecil CV error-nya dengan syarat training error dan CV error relatively close (supaya tidak overfit).

// Fitting curve untuk regularization λ:
λ kecil (slight reg.) → training error kecil, CV error besar = OVERFIT (high variance)
λ besar (large reg.) → training error besar, CV error besar = UNDERFIT (high bias)
λ optimum → training error & CV error sama-sama kecil & convergent

Ceiling Analysis — buat pipeline yang punya sub-models, evaluasi tiap komponen untuk tentukan sub-model mana yang paling butuh tuning. Misal: pipeline stock prediction = sentiment model (dari news) + structured data model. Ceiling analysis menunjukkan apakah improvement sentiment model akan memberikan lift paling besar ke overall accuracy.

Backtesting ML Signals: untuk time-series financial data, backtest pakai walk-forward — train di data historis, predict next period, roll forward. Jangan pernah pakai k-fold random shuffle di time series — itu menimbulkan look-ahead bias (model "melihat" masa depan saat training).
Quick Reference Cheat Sheet
Data Split
60% train · 20% CV · 20% test
k-Fold CV
k = 5 or 10; NOT for time series
Precision & Recall
P = TP/(TP+FP) · R = TP/(TP+FN)
F1 Score
2·P·R / (P + R)
Bias vs Variance
Under = both high · Over = CV ≫ train
AUC Interpretation
0.5 = random · 1.0 = perfect

Mini Case: Credit Default ML Model — Bank Mandiri
4 soal · Vignette-based · Pilih jawaban terbaik
Vignette
Dewi Anggraini, Senior Data Scientist di Bank Mandiri, sedang membangun model klasifikasi default untuk portofolio KPR. Dataset master berisi 50.000 nasabah selama 2019–2025, dengan 1.500 kasus default (3%). Tim memakai gradient boosting dengan regularization λ sebagai hyperparameter utama.

Info A: Dewi membagi master dataset: 30.000 nasabah untuk training, 10.000 untuk validation, dan 10.000 untuk test. Dia rutin menyesuaikan λ lalu memeriksa error rate di test set, memilih λ yang memberikan test error terkecil.

Info B: Hasil initial training: training accuracy = 99%, CV accuracy = 72%. Precision = 0.20, Recall = 0.05, F1 = 0.08. Dewi mengatakan "accuracy 99% itu sangat baik; kita siap deploy."

Info C: Analis junior Bima Saputra mengusulkan k-fold cross-validation dengan k=10 untuk memperbaiki estimasi out-of-sample error. Data timnya adalah cross-section 2019–2025 dan dia random shuffle semua observations sebelum split k-fold.

Info D: Dewi juga membangun stacked model: sub-model A (sentiment dari customer service calls, accuracy 65%) → feeds into sub-model B (gradient boosting, overall accuracy 72%). Dewi bertanya komponen mana yang seharusnya di-tune lebih dahulu untuk lift terbesar.
Soal 1 / 4
Kritik paling tepat terhadap pendekatan Dewi di Info A adalah:
B adalah jawaban yang benar. Aturan fundamental: training untuk fit parameter, validation (CV) set untuk tune hyperparameter, test set hanya untuk final evaluation — touched once. Kalau Dewi mengoptimalkan λ berdasarkan test error, dia secara efektif "bocor" informasi test set ke proses tuning, sehingga test error yang dilaporkan under-estimates true out-of-sample error. Yang benar: pakai validation set (10.000) untuk tune λ, lalu baru test final di test set. A salah karena 60:20:20 adalah standard rekomendasi CFA (dan split 30k:10k:10k = 60:20:20 untuk 50k). C salah karena gradient boosting memang pakai regularization (shrinkage, tree depth, min samples).
Soal 2 / 4
Berdasarkan Info B, kesimpulan paling akurat tentang performa model tersebut adalah:
C adalah jawaban yang benar. Tiga masalah: (1) Overfit — training error kecil, CV error jauh lebih besar (99% vs 72%). (2) Class imbalance — 3% default, jadi baseline (prediksi "no default" untuk semua) sudah kasih accuracy 97%. Accuracy 99% ≠ model bagus. (3) Recall 5% berarti dari semua actual defaulters, model hanya mendeteksi 5% (95% Type II error). Bagi bank, FN (approve kredit ke nasabah yang akan default) sangat mahal — jadi high recall critical. A salah (deploy dengan recall 5% = bencana). B salah (bukan underfit; pattern training >> CV = overfit, bukan underfit).
Soal 3 / 4
Tentang usulan Bima di Info C, penilaian yang paling tepat adalah:
B adalah jawaban yang benar. k-fold CV (k=5 atau 10) adalah standard, tetapi tidak cocok untuk time-series data — random shuffle akan membuat data masa depan muncul di training fold dan data masa lalu di validation fold, yaitu look-ahead bias. Data default KPR 2019–2025 mencakup era pre-Covid, pandemi, dan rebound — pattern makro berubah. Yang benar: walk-forward validation (train sampai tahun T, validate di tahun T+1, roll forward). A salah (random shuffle merusak time structure). C salah (k-fold bisa untuk classification maupun regression).
Soal 4 / 4
Untuk Info D (stacked model dengan sub-model), tindakan Dewi yang paling tepat adalah:
C adalah jawaban yang benar. Ceiling analysis = teknik sistematis di model pipeline untuk mengidentifikasi komponen mana yang paling membatasi overall performance. Caranya: ganti output tiap sub-model secara bergantian dengan "oracle" (ground truth). Jika accuracy melonjak saat sub-model A diganti oracle → A adalah bottleneck, prioritaskan tuning A. Jika tidak melonjak → bottleneck bukan di A, tune B (atau downstream). A dan B salah karena "selalu X dulu" adalah keputusan buta; ceiling analysis adalah cara data-driven untuk memutuskannya.
