1
Apa Itu Machine Learning? Supervised vs Unsupervised
Definisi Machine Learning

Pendekatan statistik tradisional dan teknik machine learning sama-sama menganalisis observasi untuk mengungkap proses yang mendasari data. Namun, keduanya berbeda dalam asumsi, terminologi, dan teknik. Pendekatan statistik bergantung pada asumsi fundamental dan model struktur eksplisit — misalnya, sampel dianggap berasal dari distribusi probabilitas tertentu. Asumsi a priori ini bisa gagal dalam kenyataan.

Sebaliknya, machine learning (ML) berusaha mengekstrak pengetahuan dari data dalam jumlah besar dengan lebih sedikit batasan. Tujuan algoritma ML adalah mengotomatisasi proses pengambilan keputusan dengan menggeneralisasi ("learning") dari contoh-contoh yang sudah diketahui untuk menemukan struktur mendasar dalam data. Cara paling sederhana untuk memahami ML: "temukan polanya, terapkan polanya."

ML lebih unggul dibanding pendekatan statistik tradisional (seperti regresi linear) dalam menangani masalah dengan banyak variabel (high dimensionality) atau dengan tingkat non-linearitas tinggi. Algoritma ML sangat baik dalam mendeteksi perubahan, bahkan dalam sistem yang sangat non-linear, karena dapat mendeteksi prekondisi rusaknya suatu model atau mengantisipasi probabilitas regime switch.

Tiga Kategori Utama ML

Machine learning secara luas dibagi menjadi tiga kelas teknik:

Supervised Learning: Terminologi Kunci

Dalam supervised ML, variabel dependen (Y) disebut target dan variabel independen (X) disebut features. Data berlabel (training dataset) digunakan untuk melatih algoritma agar menyimpulkan prediction rule berbasis pola. Fit model dievaluasi menggunakan labeled test data di mana target yang diprediksi (Ypredict) dibandingkan dengan target aktual (Yactual).

Supervised learning dibagi dua kategori masalah berdasarkan sifat target: jika target kontinu, maka masalahnya adalah regression; jika target kategori atau ordinal, maka masalahnya adalah classification. Contoh regression: prediksi return saham dari rasio fundamental. Contoh classification: prediksi apakah transaksi kartu kredit fraudulent atau legitimate.

Catatan Terminologi Penting

Dalam ML, istilah "regression" merujuk pada masalah dengan target kontinu — meskipun teknik ML yang dipakai bukan regresi dalam arti tradisional. Misalnya, CART dengan target kontinu disebut "regression tree" meski bukan regresi linear. Jangan bingung antara terminologi ML dan statistik tradisional.

Knowledge Check 1
Seorang analis ingin mengelompokkan 500 perusahaan ke dalam cluster berdasarkan 20 rasio fundamental tanpa menentukan berapa jumlah cluster di awal. Ini adalah contoh ML jenis apa, dan mengapa?
Unsupervised learning. Tidak ada target variabel (Y) yang ingin diprediksi — analis hanya ingin menemukan struktur (cluster) dalam data. Karena tidak ada label "kelas mana tiap perusahaan seharusnya masuk," ini bukan classification (supervised). Teknik yang cocok: k-means clustering atau hierarchical clustering.

Evaluasi Performa ML: Overfitting, Bias-Variance & Cross-Validation
Masalah Overfitting

Fleksibilitas algoritma ML memungkinkannya menangkap hubungan kompleks dan non-linear, tetapi ada harganya. Model ML yang terlalu fit terhadap training data akan tidak bisa memprediksi dengan baik pada data baru — inilah yang disebut overfitting. Model yang overfit telah "menghafal" noise dan fluktuasi random dalam training data, bukan mempelajari pola sesungguhnya.

Evaluasi performa ML harus fokus pada prediction error pada data baru, bukan pada goodness of fit pada data training. Sebuah model yang baik harus generalize — artinya mempertahankan kekuatan prediktif saat diuji pada data out-of-sample.

Dekomposisi Error: Bias vs Variance

Dataset biasanya dibagi menjadi tiga bagian yang tidak tumpang tindih: (1) training sample untuk melatih model, (2) validation sample untuk tuning dan memilih model, (3) test sample untuk menguji kemampuan prediksi pada data baru. Training dan validation sering disebut "in-sample," test disebut "out-of-sample."

Dekomposisi Total Out-of-Sample Error Eout = Bias Error + Variance Error + Base Error

Bias error → model kurang fit (underfitting) → asumsi terlalu ketat
Variance error → model terlalu fit (overfitting) → sensitif terhadap noise
Base error → randomness inheren dalam data Goal: minimize Bias + Variance secara bersamaan (bias-variance trade-off)

Fungsi linear cenderung rentan terhadap bias error dan underfitting, sementara fungsi non-linear lebih rentan terhadap variance error dan overfitting. Ada titik optimalitas kompleksitas model di mana kedua kurva error berpotongan dan total error diminimalkan — ini disebut fitting curve.

Metode Mengatasi Overfitting

Dua metode utama untuk mengurangi overfitting:

  • Regularization (complexity reduction): Batasi jumlah features dan beri penalty pada algoritma yang terlalu kompleks. Prinsip Occam's razor — solusi paling sederhana cenderung yang benar. Hanya sertakan parameter yang mengurangi out-of-sample error.
  • Cross-validation: Teknik k-fold cross-validation membagi data (selain test sample) secara random menjadi k sub-sample. Proses diulang k kali, dengan setiap sub-sample bergantian jadi validation set. Rata-rata k validation errors menjadi estimasi out-of-sample error.
Batasan K-Fold Cross-Validation

K-fold CV tidak dapat digunakan untuk time-series data karena pengacakan fold melanggar urutan temporal. Untuk time series, gunakan rolling window / walk-forward validation di mana hanya data terbaru yang digunakan untuk validasi.

Knowledge Check 2
Model ML seorang analis menghasilkan in-sample accuracy 98% tetapi out-of-sample accuracy hanya 62%. Apa diagnosis yang paling tepat, dan apa dua strategi yang bisa digunakan untuk memperbaikinya?
Diagnosis: Overfitting (high variance error). Model terlalu menyesuaikan noise training data sehingga gagal generalize. Dua strategi: (1) Regularization — kurangi kompleksitas model, misalnya memakai penalized regression atau membatasi depth decision tree. (2) Cross-validation — gunakan k-fold CV untuk mendapatkan estimasi out-of-sample error yang lebih reliable dan tuning hyperparameter berdasarkan performa validasi.

Penalized Regression: LASSO, Ridge & Elastic Net
Mengapa Penalized Regression?

Dalam konteks dataset besar dengan banyak features yang potensial berkorelasi, OLS biasa cenderung overfit. Features yang mencerminkan noise dalam training data akan masuk model dan merusak prediksi out-of-sample. Penalized regression menambahkan penalty term pada fungsi objektif sehingga feature hanya masuk model jika kontribusinya terhadap penurunan sum of squared residuals melebihi peningkatan penalty.

Formulasi LASSO (L1 Penalty)

LASSO (Least Absolute Shrinkage and Selection Operator) menambahkan penalty berupa jumlah absolute value koefisien regresi:

LASSO Objective Function Minimize: Σᵢ(Yᵢ − Ŷᵢ)² + λ Σₖ|b̂ₖ|

λ > 0 : hyperparameter (di-set sebelum training)
λ = 0 : setara dengan OLS biasa
λ ↑ : makin banyak koefisien → 0 (model lebih parsimonious) Karena L1 penalty bisa membuat koefisien persis = 0, LASSO otomatis melakukan feature selection.
Ridge Regression (L2 Penalty) dan Elastic Net

Ridge regression menggunakan penalty berupa jumlah kuadrat koefisien (L2 penalty): λ Σₖ b̂ₖ². Berbeda dengan LASSO, Ridge tidak pernah membuat koefisien persis nol — ia hanya menyusutkan koefisien mendekati nol. Ridge cocok ketika semua features dianggap relevan dan kita hanya ingin mengurangi magnitude koefisien.

Elastic Net menggabungkan L1 dan L2 penalty, memberikan fleksibilitas untuk melakukan feature selection (seperti LASSO) sekaligus menangani multicollinearity (seperti Ridge).

Aplikasi Praktis di Finance

LASSO telah digunakan untuk forecasting probabilitas default di sektor industri di mana puluhan features potensial (banyak yang collinear) direduksi menjadi kurang dari 10 variabel — penting mengingat sedikitnya observasi default (~100). Regularization juga membantu mengestimasi covariance matrix yang stabil untuk portfolio optimization, di mana return aset biasanya sangat multicollinear.

Poin Penting untuk Ujian

Lambda (λ) adalah hyperparameter — nilainya di-set sebelum training dimulai melalui proses tuning pada validation set. Penalty term hanya diaplikasikan saat training; saat evaluasi out-of-sample, hanya sum of squared residuals yang dihitung.

Knowledge Check 3
Seorang analis menjalankan LASSO dengan λ = 0.1 pada dataset 40 features. Hasilnya: 28 koefisien menjadi nol. Jika analis meningkatkan λ menjadi 0.5, apa yang paling mungkin terjadi pada jumlah features yang tersisa?
Jumlah features yang tersisa kemungkinan akan berkurang (kurang dari 12). Semakin besar λ, semakin besar penalty untuk setiap koefisien yang non-zero. Hanya features dengan kontribusi prediktif sangat kuat yang akan "bertahan" dari penalty yang lebih besar. Ini membuat model lebih parsimonious — lebih sedikit fitur, lebih tahan overfitting, tapi risiko underfitting meningkat jika λ terlalu besar.

Support Vector Machine (SVM) & K-Nearest Neighbor (KNN)
Support Vector Machine (SVM)

SVM adalah salah satu algoritma paling populer dalam machine learning. Ini adalah algoritma supervised yang kuat untuk classification, regression, dan deteksi outlier. Ide dasarnya relatif sederhana: diberikan dataset dua kelas, SVM mencari hyperplane (batas pemisah) yang memaksimalkan jarak (margin) antara kedua kelas.

Observasi terdekat ke boundary disebut support vectors — mereka yang menentukan posisi margin. Menambahkan data training yang jauh dari support vectors tidak akan mengubah boundary. Konsep kuncinya:

  • Linear SVM: Memisahkan data dengan garis lurus (2D), plane (3D), atau hyperplane (n-D). Bekerja sempurna jika data linearly separable.
  • Soft margin classification: Jika data tidak linearly separable, SVM menambahkan penalty untuk observasi yang misklasifikasi. Optimisasi menjadi trade-off antara margin yang lebih lebar dan total error penalty yang lebih rendah.
  • Non-linear SVM: Memperkenalkan batas pemisah non-linear. Bisa mengurangi misklasifikasi tapi lebih kompleks dan rentan overfitting.

Aplikasi di finance: SVM sangat cocok untuk dataset kecil-medium tapi high-dimensional (banyak features), seperti laporan keuangan perusahaan atau database kebangkrutan. SVM juga digunakan untuk mengklasifikasikan sentimen teks dari news articles dan laporan tahunan (positif vs negatif).

K-Nearest Neighbor (KNN)

KNN adalah teknik supervised learning yang paling sering digunakan untuk classification. Idenya: klasifikasikan observasi baru berdasarkan kesamaan ("nearness") dengan data yang sudah ada. Jika k = 1, observasi baru diklasifikasikan sama dengan tetangga terdekatnya. Jika k = 5, lihat 5 tetangga terdekat dan pilih kategori mayoritas.

KNN bersifat non-parametric — tidak membuat asumsi tentang distribusi data. Ini kekuatannya sekaligus kelemahannya. Tantangan kritis:

  • Definisi "similar": Pemilihan distance metric sangat penting. Metric yang salah = model yang buruk. Untuk saham, korelasi historical returns bisa dipakai sebagai ukuran kesamaan.
  • Sensitif terhadap irrelevant features: Features yang tidak relevan atau berkorelasi tinggi bisa mendistorsi distance calculation. Feature selection manual mungkin diperlukan — KNN bekerja lebih baik dengan jumlah features yang sedikit.
  • Pemilihan k (hyperparameter): k terlalu kecil → high error rate, sensitif terhadap outlier. k terlalu besar → konsep "nearest" menjadi terdilusi. Pilih k ganjil untuk menghindari ties. Teknik-teknik tertentu bisa dipakai untuk menentukan k optimal.

Knowledge Check 4
Seorang portfolio manager ingin menentukan implied credit rating (investment grade vs non-investment grade) untuk obligasi korporasi yang belum di-rating, menggunakan data rasio-rasio fundamental emiten. Algoritma mana yang lebih cocok: SVM atau KNN? Dan mengapa?
Keduanya cocok, tetapi untuk alasan berbeda. SVM cocok karena ini binary classification problem (IG vs non-IG) dan SVM bagus untuk dataset yang high-dimensional dengan banyak rasio fundamental. SVM juga tahan terhadap outlier dan features yang berkorelasi. KNN juga cocok karena obligasi dengan karakteristik serupa cenderung mendapat rating serupa — KNN memanfaatkan kesamaan ini secara langsung. Keduanya bisa digabungkan dalam voting classifier untuk hasil yang lebih robust.

Classification and Regression Tree (CART)
Cara Kerja CART

CART adalah teknik supervised ML yang membangun decision tree untuk memprediksi target variabel. Jika target kategori → classification tree. Jika target kontinu → regression tree. Struktur pohon terdiri dari:

  • Root node (node awal): Titik start, memilih feature dan cutoff value pertama.
  • Decision nodes: Setiap node mempartisi data berdasarkan satu feature dan satu cutoff value yang menghasilkan pemisahan terluas (minimum classification error atau MSE).
  • Terminal nodes (leaf): Titik akhir yang berisi prediksi — kategori mayoritas (classification) atau mean (regression).

Proses ini bersifat iteratif: setelah setiap decision node, partisi feature space menjadi semakin kecil, dan observasi dalam setiap group memiliki error yang lebih rendah. Proses berhenti saat klasifikasi error tidak berkurang signifikan dari split berikutnya.

Kekuatan Unik CART: Non-linear Interactions

Feature yang sama bisa muncul berkali-kali di level berbeda dalam tree, dalam kombinasi dengan features lain. Ini memungkinkan CART menangkap complex dependencies yang regresi linear tidak bisa. Contoh: "high profitability" hanya relevan jika P/E < 15x DAN leverage > 50% DAN sales growth > 15%. Dalam regresi linear, hubungan non-linear bersyarat seperti ini sangat sulit dimodelkan.

Menghindari Overfitting pada CART

Tanpa batasan, CART bisa perfectly learn training data — setiap observasi mendapat terminal node sendiri. Ini jelas overfitting. Metode regularisasi untuk CART:

  • Maximum depth: Batasi kedalaman pohon (jumlah level dari root ke leaf).
  • Minimum population per node: Node hanya dibagi jika jumlah observasi melebihi threshold minimum.
  • Maximum decision nodes: Batasi total jumlah node keputusan.
  • Pruning: Bangun pohon penuh terlebih dahulu, lalu pangkas bagian-bagian yang kontribusi klasifikasinya minimal.
Keunggulan Utama CART: Interpretability

Dibanding algoritma lain yang sering dianggap "black box" (sulit memahami logika di balik prediksi), CART menghasilkan visual tree yang sangat mudah dipahami dan dikomunikasikan ke klien, investment committee, dan non-technical stakeholders. Ini menjadi keunggulan besar dalam konteks investment management di mana transparansi keputusan investasi sangat penting.

Knowledge Check 5
Sebuah model CART untuk deteksi fraud pada laporan keuangan menghasilkan decision tree dengan depth 25 level dan 300 terminal nodes. In-sample accuracy 99.8% tapi out-of-sample accuracy hanya 58%. Apa masalahnya dan sebutkan dua solusi spesifik.
Masalah: Severe overfitting. Tree terlalu dalam (25 level) dan terlalu banyak terminal nodes (300) — model menghafal training data termasuk noise-nya. Dua solusi: (1) Regularization — batasi max depth (misalnya 5-8 level) dan/atau set minimum population per node (misalnya 50 observasi). (2) Pruning — bangun tree penuh lalu pangkas cabang-cabang yang kontribusi akurasinya minimal pada validation set. Kedua metode mengurangi kompleksitas dan meningkatkan generalization.
6
Ensemble Learning: Voting, Bagging, Random Forest & Gradient Boosting
Prinsip Dasar Ensemble Learning

Mengapa mengandalkan satu model jika kita bisa menggabungkan prediksi dari banyak model? Setiap model individu memiliki error rate tertentu dan membuat prediksi yang noisy. Tapi dengan mengambil rata-rata (atau majority vote) dari banyak prediksi, noise cenderung saling meniadakan dan prediksi konvergen ke akurasi yang lebih tinggi. Inilah ensemble learning — teknik yang secara konsisten menghasilkan prediksi lebih akurat dan lebih stabil dibanding model tunggal terbaik.

Voting Classifiers (Heterogeneous Ensemble)

Pendekatan paling sederhana: latih beberapa algoritma berbeda (misalnya SVM, KNN, CART) pada dataset yang sama, lalu buat prediksi akhir berdasarkan majority vote. Contoh: jika SVM dan KNN memprediksi "outperformance" tapi CART memprediksi "underperformance," voting classifier memilih outperformance (2 vs 1). Kunci keberhasilan: diversitas dalam pilihan algoritma. Ada jumlah model optimal — terlalu banyak justru bisa overfit.

Bootstrap Aggregating (Bagging)

Bagging menggunakan algoritma yang sama tetapi dengan data training yang berbeda. Caranya: dari dataset awal, buat n dataset baru via random sampling with replacement (bootstrap). Setiap "bag" digunakan untuk melatih model terpisah. Prediksi akhir = majority vote (classification) atau average (regression). Bagging sangat efektif untuk meningkatkan stabilitas prediksi dan melindungi dari overfitting.

Random Forest

Random forest = koleksi banyak decision trees yang dilatih via bagging, dengan tambahan: di setiap node, hanya subset random m features (di mana m < n total features) yang dipertimbangkan untuk splitting. Ini menambah diversitas antar-tree dan lebih jauh mengurangi variance.

Klasifikasi akhir dalam random forest = majority vote dari semua tree — implementasi machine learning dari "wisdom of crowds." Proses ini mengurangi variance dan melindungi dari overfitting karena error saling meniadakan antar tree yang sedikit berbeda. Namun, kelemahan utama random forest adalah kurangnya interpretability — sulit menjelaskan mengapa prediksi tertentu dibuat, sehingga dianggap "black box."

Gradient Boosting

Berbeda dengan bagging yang membangun tree secara paralel dan independen, gradient boosting membangun tree secara sekuensial. Setiap tree baru fokus memperbaiki error (residual) dari tree sebelumnya. Prosesnya:

  • Latih tree pertama pada data asli.
  • Hitung residual (error) dari tree pertama.
  • Latih tree kedua pada residual tersebut — fokus pada observasi yang salah diprediksi.
  • Ulangi sampai stopping criterion tercapai.

Gradient boosting sering menghasilkan akurasi lebih tinggi dari random forest, tapi lebih rentan terhadap overfitting karena setiap tree baru bergantung pada tree sebelumnya. Tuning hyperparameter (learning rate, jumlah trees, max depth) sangat kritis.

Perbandingan Kritis: Bagging vs Boosting

Bagging (Random Forest): Parallel training, trees independen, primarily reduce variance. Lebih robust, kurang prone overfitting.

Boosting (Gradient Boosting): Sequential training, trees dependent, reduce both bias and variance. Potensial akurasi lebih tinggi tapi membutuhkan tuning lebih hati-hati.

Aplikasi Random Forest di Finance

Random forest telah digunakan secara luas dalam factor-based investment strategies untuk asset allocation dan investment selection, prediksi keberhasilan IPO (oversubscription, first-day return), deteksi fraud kartu kredit (mampu menemukan 82% kasus fraud di dataset dengan hanya 0.17% fraud rate), dan prediksi credit default perusahaan kecil-menengah — di mana ia jauh mengungguli model regresi linear tradisional.

Knowledge Check 6
Mengapa random forest cenderung lebih robust terhadap overfitting dibanding single CART? Identifikasi dua mekanisme spesifik yang berkontribusi.
Dua mekanisme: (1) Bagging — setiap tree dilatih pada bootstrap sample yang berbeda dari data asli. Ini berarti setiap tree melihat variasi data yang sedikit berbeda, sehingga overfitting terhadap noise di satu subset data tertentu dikurangi ketika prediksi diagregasi. (2) Random feature subset — di setiap node, hanya sebagian features (m < n) yang dipertimbangkan. Ini mengurangi korelasi antar-tree (jika satu feature dominan, tidak semua tree akan menggunakannya di root node) dan meningkatkan diversitas. Kedua mekanisme bekerja sama untuk mengurangi variance error secara signifikan.

Latihan Soal — Item Set Vignette
Vignette: PT Bank Rakyat Indonesia & Model Prediksi Default UMKM
6 soal · Tingkat kesulitan CFA Exam
Vignette
Dewi Kartika, Head of Credit Analytics di PT Bank Rakyat Indonesia Tbk (BBRI), memimpin tim yang sedang membangun model prediksi default untuk kredit UMKM. Dataset terdiri dari 120.000 debitur UMKM dengan 42 features (rasio keuangan, sektor usaha, lokasi, usia usaha, riwayat pembayaran, dll.) dan target binary: default (1) atau performing (0). Tingkat default historis adalah 4.2%.

Tim Dewi menguji empat pendekatan:

Model 1 — OLS Logistic Regression: Menggunakan semua 42 features. In-sample accuracy 91%, out-of-sample accuracy 72%. Banyak koefisien memiliki p-value > 0.10. VIF beberapa variabel melebihi 10.

Model 2 — LASSO (λ = 0.08): Mereduksi features dari 42 menjadi 15. Out-of-sample accuracy 79%. Koefisien yang tersisa memiliki tanda yang konsisten dengan teori kredit.

Model 3 — CART (max depth = 15, min population = 5): In-sample accuracy 97%, out-of-sample accuracy 68%. Tree menghasilkan 892 terminal nodes.

Model 4 — Random Forest (300 trees, max depth = 8, m = 7): Out-of-sample accuracy 85%. Feature importance analysis menunjukkan riwayat pembayaran, debt-service coverage ratio, dan usia usaha sebagai top-3 predictors.

Dewi juga mempertimbangkan gradient boosting sebagai Model 5 dan KNN (k = 5) sebagai Model 6 untuk evaluasi lebih lanjut. Tim internal risk management BBRI meminta Dewi mempresentasikan logika model ke Dewan Komisaris yang mayoritas non-technical.
Soal 1 / 6
VIF (Variance Inflation Factor) beberapa variabel dalam Model 1 melebihi 10. Masalah ini paling tepat ditangani oleh metode mana?
B adalah jawaban yang benar. VIF > 10 mengindikasikan multicollinearity parah di antara features. LASSO (L1 penalty) secara otomatis melakukan feature selection — koefisien features yang kurang penting atau redundan (karena collinear) didorong ke nol. Dibandingkan Model 2 (LASSO), jika Dewi menggunakan Ridge regression dengan λ yang sama, perbedaan paling signifikan yang diharapkan adalah:
