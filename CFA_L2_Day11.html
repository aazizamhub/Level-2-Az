Theory · 30 min
1
ML Fundamentals: Supervised vs Unsupervised & Overfitting

Machine learning (ML) secara garis besar dibagi tiga: supervised (ada target Y + features X, data berlabel), unsupervised (hanya X, tanpa target — cari struktur tersembunyi), dan deep learning/reinforcement learning.

AspekSupervisedUnsupervised
DataLabeled (Y + X)Unlabeled (X only)
TujuanPrediksi target (regression/classification)Temukan struktur (clustering/dimension reduction)
ContohPrediksi return saham, deteksi fraudGrouping perusahaan, PCA

Overfitting = model terlalu "menghafal" training data termasuk noise-nya, sehingga jelek di data baru. Tanda: in-sample error rendah tapi out-of-sample error tinggi. Dua cara utama atasi: (1) regularization — batasi kompleksitas model via penalty, (2) cross-validation — k-fold CV, bagi data jadi k subset, rotasi validasi.

Total out-of-sample error = Bias error + Variance error + Base error
// Bias↑ → underfitting. Variance↑ → overfitting.
// Goal: minimize keduanya secara bersamaan.
⚠ Exam Trap

k-fold CV tidak bisa dipakai untuk time-series data karena asumsi independensi antar fold dilanggar. Untuk time series, gunakan rolling window (walk-forward) validation.

2
Penalized Regression: LASSO, Ridge & Elastic Net

Ketika features banyak dan saling berkorelasi, OLS biasa cenderung overfit. Penalized regression menambahkan penalty term pada sum of squared residuals supaya hanya features yang benar-benar penting yang masuk model.

// LASSO (L1 penalty):
min Σ(Yᵢ − Ŷᵢ)² + λ Σ|b̂ₖ|
// Ridge (L2 penalty):
min Σ(Yᵢ − Ŷᵢ)² + λ Σ b̂ₖ²
// Elastic Net = kombinasi L1 + L2

Lambda (λ) adalah hyperparameter — di-set sebelum training. λ = 0 → sama dengan OLS. Makin besar λ → makin banyak koefisien yang shrink ke nol (terutama LASSO). LASSO otomatis melakukan feature selection karena bisa "mematikan" variabel (koefisien = 0).

Key Insight: LASSO sangat populer di finance karena sering berhadapan dengan banyak fitur yang collinear (misalnya rasio-rasio fundamental). LASSO membantu memilih variabel yang paling prediktif dan menghasilkan model parsimonious yang lebih baik out-of-sample.
⚠ Exam Trap

Penalty term hanya dipakai saat training. Setelah model jadi, evaluasi out-of-sample hanya pakai sum of squared residuals tanpa penalty. Jangan campur tahap training dengan tahap testing!

3
SVM & KNN — Algoritma Klasifikasi

Support Vector Machine (SVM): Cari hyperplane yang memisahkan dua kelas dengan maximum margin. Titik-titik terdekat ke boundary disebut support vectors. Kalau data tidak linearly separable → gunakan soft margin (toleransi misklasifikasi dengan penalty) atau non-linear SVM.

K-Nearest Neighbor (KNN): Klasifikasikan observasi baru berdasarkan mayoritas dari k tetangga terdekat. Non-parametric, nggak perlu asumsi distribusi. Tapi sensitif terhadap: (1) pilihan k — terlalu kecil → overfit, terlalu besar → diluted, (2) distance metric, (3) fitur yang irrelevant/correlated.

AspekSVMKNN
KekuatanBagus untuk dataset kecil-medium, high-dimensional; tahan outlierSimpel, intuitif, non-parametric; langsung multi-class
KelemahanSulit interpretasi; non-linear SVM rawan overfitSensitif terhadap irrelevant features; harus pilih k dan distance metric
AplikasiPrediksi kebangkrutan, sentimen analisis teksCredit rating assignment, bond index creation
4
CART — Classification & Regression Tree

CART membangun decision tree dengan cara mempartisi feature space secara iteratif. Setiap node memilih feature + cutoff value yang memaksimalkan pemisahan data (minimkan classification error atau MSE).

Struktur: Root node → Decision nodes → Terminal nodes (leaf). Fitur yang sama bisa muncul berkali-kali di level berbeda — CART bisa menangkap interaksi kompleks non-linear yang linear regression gagal tangkap.

Regularization CART: Tanpa batasan, CART bisa "menghafal" semua training data (perfect fit → overfit). Solusi: batasi max depth tree, min population per node, max decision nodes, atau pruning (potong cabang yang kontribusinya kecil).

Key Insight: Keunggulan utama CART vs algoritma lain: interpretability. Output berupa visual tree yang mudah dijelaskan ke klien dan investment committee — bukan "black box" seperti SVM atau random forest.
5
Ensemble Learning: Random Forest & Gradient Boosting

Daripada mengandalkan satu model, ensemble learning menggabungkan prediksi banyak model untuk mengurangi noise dan meningkatkan akurasi. Dua pendekatan utama:

1. Voting Classifiers: Gabungkan beberapa algoritma berbeda (SVM + KNN + CART), ambil keputusan berdasarkan suara mayoritas.

2. Bagging (Bootstrap Aggregating): Buat n dataset baru via random sampling with replacement, latih algoritma yang sama di masing-masing, lalu agregasi prediksi. Mengurangi variance dan melindungi dari overfitting.

Random Forest: Koleksi banyak decision tree yang dilatih via bagging + random feature subset di setiap node. Hyperparameter: jumlah tree, max depth, min population, jumlah fitur subset (m < n). Klasifikasi final = majority vote. Sangat powerful tapi "black box".

Gradient Boosting: Membangun tree secara sequential — setiap tree baru fokus memperbaiki error tree sebelumnya. Lebih prone overfitting daripada random forest tapi sering menghasilkan akurasi lebih tinggi kalau di-tune dengan baik.

⚠ Exam Trap

Random forest = bagging (parallel, reduce variance). Gradient boosting = sequential (reduce bias + variance, tapi rawan overfit). Jangan tukar keduanya di ujian — mereka fundamentally berbeda meski sama-sama ensemble.

Quick Reference Cheat Sheet
LASSO Penalty
λ Σ|b̂ₖ| → shrinks to 0 → feature selection
Ridge Penalty
λ Σ b̂ₖ² → shrinks toward 0, never exactly 0
SVM Core Idea
Max-margin hyperplane; support vectors
KNN Hyperparameter
k = # neighbors; odd k avoids ties
Random Forest
Bagging + random feature subset → low variance
Overfitting Fix
Regularization + k-fold cross-validation

Mini Vignette — PT Telkom Indonesia & Prediksi Churn Pelanggan
4 soal · Supervised Learning
Vignette
Sinta Maharani, data scientist di divisi analytics PT Telkom Indonesia Tbk (TLKM), sedang mengembangkan model prediksi churn pelanggan IndiHome. Dataset terdiri dari 50.000 pelanggan dengan 35 features (durasi berlangganan, paket, tagihan rata-rata, jumlah komplain, usage patterns, dll.) dan target binary: churn (1) atau retain (0).

Sinta pertama mencoba OLS linear regression dengan semua 35 features. Hasilnya: in-sample R² = 0.92 namun out-of-sample R² turun drastis ke 0.41. Banyak koefisien yang signifikan secara statistik tapi tanda-nya berlawanan dengan intuisi bisnis.

Rekan Sinta, Budi Prasetyo, menyarankan tiga pendekatan alternatif:
• Model A: LASSO regression dengan λ = 0.05 — menghasilkan model dengan hanya 12 features (23 features koefisiennya menjadi 0)
• Model B: Random forest dengan 500 trees, max depth 10, dan random feature subset m = 6 di setiap node
• Model C: Single CART decision tree tanpa batasan depth

Setelah evaluasi, Model A menghasilkan out-of-sample accuracy 78%, Model B menghasilkan 84%, dan Model C menghasilkan in-sample accuracy 99% namun out-of-sample hanya 65%.
Soal 1 / 4
Masalah utama yang dialami Sinta dengan OLS regression menggunakan 35 features paling tepat diidentifikasi sebagai:
B adalah jawaban yang benar. In-sample R² sangat tinggi (0.92) tapi out-of-sample R² jauh lebih rendah (0.41) — ini adalah tanda klasik overfitting (high variance error). Model menghafal noise di training data. Koefisien yang "salah tanda" juga mengindikasikan multicollinearity yang parah di antara 35 features, yang membuat estimasi OLS tidak stabil. A salah karena in-sample fit-nya sudah tinggi (bukan underfitting). C salah karena masalahnya bukan bias (model fit in-sample baik) tapi variance.
Soal 2 / 4
LASSO regression (Model A) mampu mengeliminasi 23 dari 35 features karena:
A adalah jawaban yang benar. LASSO menggunakan L1 penalty (absolute value koefisien). Karena sifat geometrinya, L1 penalty bisa membuat koefisien persis nol — berbeda dengan Ridge (L2) yang hanya mendekati nol tapi tidak pernah tepat nol. Ini yang membuat LASSO menjadi alat feature selection otomatis. B mendeskripsikan Ridge, bukan LASSO. C salah karena cross-validation adalah teknik evaluasi, bukan mekanisme yang menghilangkan features.
Soal 3 / 4
Performa Model C (single CART, in-sample 99% tapi out-of-sample 65%) dibandingkan Model B (random forest, out-of-sample 84%) paling baik dijelaskan oleh:
C adalah jawaban yang benar. CART tanpa regularization (unrestricted depth) akan "menghafal" training data → in-sample 99% tapi out-of-sample jelek (65%). Random forest mengatasi ini dengan: (1) bagging — training data yang berbeda untuk setiap tree, dan (2) random feature subset — setiap node hanya pilih dari m features random. Kedua mekanisme ini mengurangi variance (noise cancelling) dan melindungi dari overfitting. A salah karena masalah CART di sini adalah variance bukan bias. B salah karena random forest adalah supervised, bukan unsupervised.
Soal 4 / 4
Jika Sinta perlu mempresentasikan model prediksi churn kepada direksi Telkom yang tidak memiliki background data science, algoritma mana yang paling tepat dipilih dari sudut interpretability?
B adalah jawaban yang benar. Keunggulan utama CART adalah interpretability. Hasilnya berupa visual tree: "jika customer komplain > 3 kali DAN tagihan > Rp500rb DAN durasi < 6 bulan → prediksi churn." Ini sangat mudah dipahami oleh non-technical audience. Dengan menambahkan regularization (max depth, pruning), CART bisa menghindari overfitting sambil tetap interpretable. A salah karena random forest justru disebut "black box" — sulit menjelaskan kenapa prediksi tertentu dibuat. C salah karena SVM dengan banyak features menjadi sulit divisualisasikan (hyperplane di 35 dimensi tidak intuitif).
