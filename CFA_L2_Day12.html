Theory · 30 min
1
Principal Components Analysis (PCA) — Dimension Reduction

Ketika dataset punya puluhan atau ratusan features yang saling berkorelasi, model jadi kompleks dan rentan overfitting. PCA mengatasi ini dengan mentransformasi features asli menjadi principal components — variabel komposit baru yang saling tidak berkorelasi (uncorrelated).

PCA bekerja dengan dua konsep kunci: eigenvectors (mendefinisikan arah PC baru sebagai kombinasi linear features asli) dan eigenvalues (menunjukkan proporsi total variance yang dijelaskan oleh tiap PC). PC1 menjelaskan variance terbesar, PC2 menjelaskan variance terbesar berikutnya, dst.

PC1 = w₁₁X₁ + w₁₂X₂ + ... + w₁ₙXₙ
// Eigenvector dengan eigenvalue terbesar → paling banyak menjelaskan variance.
// Gunakan scree plot untuk tentukan jumlah PC yang cukup (biasanya 85-95% total variance).

Dalam praktik investasi, PCA sering dipakai untuk mengidentifikasi faktor-faktor utama yang menggerakkan harga aset. Misalnya, dari 500 saham large-cap, cukup 3 PC yang menjelaskan ~90% variance return.

Key Insight: PCA mengurangi dimensi data sebelum training model lain (supervised maupun unsupervised). Hasilnya: model lebih cepat, mengurangi overfitting, dan lebih mudah diinterpretasi. Kelemahannya: PC adalah "black box" — sulit diberi label bermakna karena merupakan kombinasi banyak features.
2
K-Means vs Hierarchical Clustering

Clustering mengelompokkan observasi ke dalam cluster berdasarkan kesamaan (similarity). Tujuannya: cohesion (observasi dalam cluster mirip) dan separation (observasi antar-cluster berbeda). Distance measure (biasanya Euclidian) digunakan untuk mengukur kesamaan.

AspekK-MeansHierarchical
Jumlah cluster (k)Harus ditentukan sebelum run (hyperparameter)Tidak perlu — algoritma membuat hierarki lengkap
ProsesIteratif: assign → recalculate centroid → reassign sampai konvergenAgglomerative (bottom-up) atau Divisive (top-down)
KecepatanSangat cepat, cocok dataset sangat besarLebih lambat (computationally intensive)
KeunggulanEfisien untuk big data, mudah diimplementasiFleksibel — bisa periksa berbagai level granularitas via dendrogram
KelemahanHasil bisa tergantung posisi centroid awalTidak scalable untuk dataset sangat besar

Dendrogram adalah diagram pohon yang memvisualisasikan hierarki cluster. Sumbu Y = distance, semakin pendek dendrite = cluster makin mirip. Analyst bisa "memotong" dendrogram di level yang diinginkan untuk memilih jumlah cluster optimal.

⚠ Exam Trap

K-means: hasil akhir bisa berubah tergantung posisi centroid awal yang random. Solusinya: jalankan algoritma beberapa kali dengan centroid awal berbeda, pilih yang paling berguna. Hierarchical: agglomerative (bottom-up) cocok untuk menemukan cluster kecil; divisive (top-down) lebih baik untuk cluster besar karena memulai dari global structure.

3
Neural Networks & Deep Learning

Neural networks (NNs) terdiri dari tiga jenis layer: input layer (menerima features), hidden layer(s) (tempat "learning" terjadi), dan output layer (menghasilkan prediksi). Setiap node di hidden layer memiliki dua fungsi: summation operator (menjumlahkan weighted inputs) dan activation function (transformasi non-linear).

// Contoh activation function — ReLU:
f(x) = max(0, x)
// Sigmoid function: output range (0,1) — cocok untuk classification.
// Training: forward propagation → compare to actual → backward propagation → update weights.

Deep Neural Networks (DNNs) = NN dengan ≥2 hidden layers (bisa 20+). DNN unggul di task kompleks: image recognition, speech recognition, NLP, dan option pricing. Hyperparameters DNN: jumlah hidden layers, nodes per layer, jenis activation function, dan learning rate.

New weight = Old weight − (Learning rate) × ∂(Total error) / ∂(Old weight)
// Learning rate = hyperparameter yang mengontrol besarnya penyesuaian.
// Terlalu besar → gagal konvergen. Terlalu kecil → training sangat lambat.
Key Insight: NN/DNN sangat powerful untuk menangkap non-linear relationships, tapi trade-off-nya: (1) "black box" — sulit diinterpretasi, (2) butuh data sangat besar, dan (3) komputasi intensif. Jadi NN bukan selalu pilihan terbaik — untuk dataset kecil atau ketika interpretability penting, model lain lebih cocok.
4
Natural Language Processing (NLP) & Sentiment Analysis

NLP memungkinkan mesin memproses dan menganalisis data teks. Dalam finance, NLP digunakan untuk mengekstrak sentiment dari earnings calls, press releases, dan financial filings. Pipeline dasar NLP:

1. Text Cleansing: hapus HTML tags, punctuation, numbers, extra whitespace.
2. Tokenization: pecah teks menjadi tokens (kata-kata individual).
3. Normalization: lowercasing, remove stop words ("the", "is", "a"), stemming/lemmatization (mengubah kata ke bentuk dasar).
4. Bag-of-Words (BOW): kumpulan distinct tokens dari seluruh corpus — tidak memperhatikan urutan kata.
5. Document Term Matrix (DTM): matriks (baris = dokumen, kolom = token, sel = frekuensi) — data terstruktur siap untuk ML.

// Stemming: "analyzed" → "analyz", "analyzing" → "analyz" (rule-based)
// Lemmatization: "analyzed" → "analyze", "analyzing" → "analyze" (morphological root)
// BOW → DTM → siap training supervised ML (e.g., sentiment classification)
⚠ Exam Trap

BOW tidak menangkap urutan atau posisi kata — hanya frekuensi. Ini berarti konteks bisa hilang. Stemming lebih sederhana tapi hasilnya belum tentu linguistically correct (misalnya "analyz" bukan kata asli). Lemmatization lebih akurat tapi computationally lebih mahal.

Quick Reference Cheat Sheet
PCA Goal
Reduce dimensions, retain 85-95% variance
K-Means Key
k = hyperparameter, iterasi sampai konvergen
Hierarchical
Agglom = bottom-up, Divisive = top-down
NN Training
Forward prop → error → backprop → update
DNN vs NN
DNN = ≥2 hidden layers (shallow NN = 1)
NLP Pipeline
Cleanse → tokenize → normalize → BOW → DTM

Mini Case: Analytic Team PT Mandiri Sekuritas
4 soal · Vignette-based · Pilih jawaban terbaik
Vignette
Andi Pratama, head of quantitative research di PT Mandiri Sekuritas, ditugaskan untuk memodernisasi proses analisis portofolio. Tim Andi sedang mengerjakan tiga proyek:

Proyek 1 — Equity Clustering: Tim ingin mengelompokkan 200 saham IDX berdasarkan 15 financial features (P/E, P/B, ROE, leverage, revenue growth, dll.) tanpa menggunakan klasifikasi sektoral konvensional. Mereka memilih k-means clustering dengan k = 8, tetapi hasil clustering berubah setiap kali algoritma dijalankan.

Proyek 2 — Risk Factor Model: Tim memiliki dataset return 200 saham dengan 50 features fundamental dan teknikal. Banyak features saling berkorelasi tinggi (multicollinearity). Tim menerapkan PCA dan menemukan bahwa 4 principal components menjelaskan 88% total variance. Namun, saat mempresentasikan ke manajemen, mereka kesulitan menjelaskan "apa arti" setiap PC.

Proyek 3 — Sentiment Analysis: Tim mengunduh 10.000 transkrip earnings calls perusahaan IDX. Mereka membangun pipeline NLP: text cleansing → tokenization → normalization → bag-of-words → document term matrix. Logistic regression digunakan untuk mengklasifikasikan sentiment (positif/negatif) berdasarkan DTM. Model menunjukkan accuracy 72%.
Soal 1 / 4
Hasil clustering Proyek 1 yang berubah-ubah setiap kali dijalankan paling mungkin disebabkan oleh:
A adalah jawaban yang benar. K-means dimulai dengan memilih posisi centroid awal secara random. Posisi awal yang berbeda bisa menghasilkan konvergensi ke cluster assignments yang berbeda. Solusinya: jalankan algoritma beberapa kali dengan centroid awal berbeda, kemudian pilih clustering yang paling berguna berdasarkan business purpose. B salah karena k-means bekerja baik bahkan dengan dataset sangat besar (ratusan juta observasi). C salah karena k-means adalah unsupervised — memang tidak memerlukan target variable.
Soal 2 / 4
Kesulitan tim dalam menjelaskan arti setiap PC pada Proyek 2 merupakan kelemahan utama PCA yang paling tepat dideskripsikan sebagai:
B adalah jawaban yang benar. Kelemahan utama PCA: setiap PC adalah kombinasi linear dari semua features asli. Karena itu, PC sulit diberi label bermakna — misalnya, "PC1" tidak bisa langsung disebut "value factor" atau "growth factor." End user mungkin melihat PCA sebagai "black box." A salah karena PCA memang membuang sebagian variance (yang kecil), tapi ini trade-off yang disengaja — bukan kelemahan utama yang ditanyakan. C salah karena PCA memang cocok untuk data numerik seperti features fundamental/teknikal.
Soal 3 / 4
Dalam pipeline NLP Proyek 3, tahap "normalization" yang paling meningkatkan efisiensi training ML model adalah:
C adalah jawaban yang benar. Stop word removal menghilangkan kata-kata umum ("the", "is", "a") yang tidak bermakna untuk analisis. Stemming/lemmatization mengubah kata ke bentuk dasar (misalnya "analyzed", "analyzing" → "analyz") sehingga mengurangi repetisi dan data sparseness. Keduanya mengecilkan dimensi DTM secara signifikan → ML training lebih efisien. A dan B adalah tahap yang mendahului normalization dan penting, tapi pertanyaannya spesifik tentang tahap yang paling meningkatkan efisiensi training.
Soal 4 / 4
Jika Andi ingin menggunakan deep neural network untuk memperbaiki sentiment model di Proyek 3, keuntungan DNN dibanding logistic regression yang digunakan saat ini adalah:
A adalah jawaban yang benar. Keunggulan utama DNN: kemampuan menangkap non-linear relationships dan interaksi kompleks antar-features berkat arsitektur multi-layer dengan activation functions non-linear. Logistic regression terbatas pada hubungan linear. B salah karena DNN justru membutuhkan lebih banyak data untuk training yang efektif. C salah karena DNN adalah "black box" — kurang interpretable dibanding logistic regression.
