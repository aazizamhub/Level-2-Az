Kenapa Logistic Regression? Masalah Linear Probability Model

Sejauh ini kita membahas regression dengan continuous dependent variable. Tapi dalam banyak analisis keuangan, kita ingin memprediksi outcomes yang bersifat binary — ya/tidak, terjadi/tidak terjadi:

  • Apakah perusahaan ini akan default dalam 12 bulan?
  • Apakah emiten akan melakukan share repurchase tahun ini?
  • Apakah ETF ini akan menjadi "winning fund" bulan depan?
  • Apakah sentimen press release ini positif atau negatif?
1.1 Masalah Linear Probability Model (LPM)

Bayangkan kita coba langsung pakai OLS dengan Y = 1 jika event terjadi, Y = 0 jika tidak:

Yᵢ = b₀ + b₁X₁ᵢ + b₂X₂ᵢ + b₃X₃ᵢ + εᵢ, di mana Y ∈ {0,1}

Ada dua masalah fundamental dengan pendekatan ini:

  • Predicted values di luar [0,1]: Bergantung pada koefisien dan nilai X, Ŷ bisa menghasilkan angka di atas 1.0 atau di bawah 0 — tidak valid sebagai probabilitas. Probabilitas tidak bisa >100% atau <0%.
  • Hubungan non-linear: Logika menyarankan bahwa ketika perusahaan sudah sangat sehat (DER sangat rendah), tambahan penurunan DER hampir tidak mengubah probabilitas default. Begitu juga ketika perusahaan sudah hampir pasti default. Hubungannya berbentuk S-curve, bukan garis lurus.
📊 Visualisasi — LPM vs Logit Model

Linear Probability Model (Panel A): Garis lurus yang bisa menembus 0 dan 1. Tidak valid untuk probability.

Logit Model (Panel B): S-curve (sigmoidal) yang dimulai mendekati 0 untuk X sangat rendah, naik perlahan, lalu cepat di tengah, kemudian mendatar mendekati 1 untuk X sangat tinggi. Selalu antara 0 dan 1.

Kurva sigmoidal juga realistis secara ekonomi: di ujung-ujung distribusi, perubahan X kecil dampaknya terhadap probabilitas, tapi di tengah (sekitar P = 0.5), perubahan X paling impactful.

2
Model Logistik — Odds, Log-Odds, dan Sigmoid

Untuk mengatasi masalah LPM, kita aplikasikan logistic transformation — mentransformasi probability P ke dalam log-odds yang rangenya tidak terbatas (−∞ sampai +∞), sehingga bisa diregresikan secara linear.

2.1 Tiga Konsep Terhubung
Step 1: Dari Probability ke Odds Odds = P / (1 − P) Range: 0 sampai +∞ Contoh: P = 0.75 → Odds = 0.75/0.25 = 3.0 (peluang 3 berbanding 1) Contoh: P = 0.50 → Odds = 1.0 (even odds)
Step 2: Dari Odds ke Log-Odds (Logit) Log-odds = ln[P / (1 − P)] Range: −∞ sampai +∞ → bisa di-regresi linear! P = 0.5 → Log-odds = 0 P > 0.5 → Log-odds > 0 (positif) P < 0.5 → Log-odds < 0 (negatif)
Model Logistik Lengkap ln[P / (1−P)] = b₀ + b₁X₁ + b₂X₂ + b₃X₃ + ε ← diestimasi
P = 1 / {1 + exp[−(b₀ + b₁X₁ + b₂X₂ + b₃X₃)]} ← dikonversi Selalu menghasilkan P antara 0 dan 1 ✓
💡 Intuisi Sigmoid

Formula P = 1/(1 + e^(−z)) di mana z = linear predictor menghasilkan fungsi sigmoidal:

— Ketika z → +∞: e^(−z) → 0, sehingga P → 1/(1+0) = 1

— Ketika z = 0: e^0 = 1, sehingga P = 1/(1+1) = 0.5

— Ketika z → −∞: e^(−z) → +∞, sehingga P → 0

Ini berarti model selalu menghasilkan P antara 0 dan 1, dan slope kurva (marginal effect) paling besar di P = 0.5 serta paling kecil di ujung-ujung distribusi.

3
Estimasi MLE dan Interpretasi Koefisien
3.1 Maximum Likelihood Estimation (MLE)

Logistic regression tidak menggunakan OLS (least squares) karena tidak ada closed-form solution. Sebagai gantinya, koefisien diestimasi dengan MLE (Maximum Likelihood Estimation) — mencari nilai b₀, b₁, ..., bₖ yang membuat data yang kita observasi "paling mungkin" terjadi.

Secara teknis: MLE memaksimalkan likelihood function — produk dari probabilitas tiap observasi diklasifikasikan dengan benar. Untuk binary outcome (binomial distribution), ini setara dengan memaksimalkan log-likelihood. Software menghitungnya secara iteratif menggunakan algoritma seperti Newton-Raphson.

⚠ MLE Properties — Yang Perlu Diketahui untuk Exam

MLE menghasilkan estimator yang consistent (semakin besar n → semakin mendekati nilai benar) dan asymptotically normal (distribusi sampling mendekati normal untuk sampel besar). Ini mengapa hypothesis test di logit menggunakan z-statistic, bukan t-statistic. Untuk sampel besar, z-statistic dan t-statistic memberikan hasil yang sangat mirip.

3.2 Interpretasi Koefisien — Paling Sering Diuji
🚨 Interpretasi yang Benar — Exam-Critical

Koefisien bⱼ dalam logit model = perubahan log-odds per unit kenaikan Xⱼ, holding all other variables constant.

Ini BUKAN perubahan probabilitas P. Karena hubungannya non-linear, perubahan probabilitas per unit Xⱼ berbeda-beda tergantung nilai semua X saat ini.

Perbandingan:
OLS: b₁ = 0.5 artinya "Y naik 0.5 per unit X₁ naik, ceteris paribus"
Logit: b₁ = 0.5 artinya "log-odds naik 0.5 per unit X₁ naik, ceteris paribus" — untuk tahu dampaknya terhadap P, perlu kalkulasi terpisah

Dari logit model share repurchase (curriculum), koefisien DE = −0.3186. Interpretasi: kenaikan 1 unit DE menurunkan log-odds repurchase sebesar 0.3186 — holding CASH dan NPM constant. Ini konsisten dengan intuisi ekonomi: perusahaan yang lebih leverage (DER tinggi) cenderung tidak buyback karena lebih perlu jaga cash.

3.3 Interpretasi Intercept

Intercept b₀ adalah log-odds dari event ketika semua independent variables = 0. Untuk mengkonversi ke probabilitas:

Probability dari Intercept (semua X = 0) P(Y=1 | X₁=X₂=...=Xₖ=0) = 1 / (1 + e^(−b₀)) Contoh: b₀ = −0.4738 → P = 1/(1+e^0.4738) = 1/(1+1.606) = 38.37%
Knowledge Check 3.1 — Interpretasi Koefisien Logit

Dari model ETF winning fund (Logistic Regression 1): koefisien price_sales = 0.3432 (z=4.416, p=0.000). Bagaimana interpretasi yang tepat untuk koefisien ini?

Kenaikan price-to-sales ratio (P/S) sebesar 1 unit meningkatkan log-odds ETF menjadi "winning fund" sebesar 0.3432, holding semua variabel lain constant. Ini bukan berarti probabilitas naik 34.32% — itu salah total. Koefisien positif menunjukkan bahwa ETF dengan P/S lebih tinggi cenderung lebih mungkin menjadi winning fund, tapi besaran dampaknya terhadap probabilitas tergantung pada nilai semua variabel lain untuk ETF tersebut.
4
Marginal Effect — Prosedur Dua Langkah

Untuk mendapat dampak aktual perubahan Xⱼ terhadap probabilitas, kita perlu menghitung marginal effect secara eksplisit. Karena hubungannya non-linear, marginal effect bervariasi per observasi.

4.1 Marginal Effect pada "Average Observation"

Cara paling umum: hitung marginal effect pada nilai rata-rata semua variabel — ini memberikan dampak "tipikal" dalam dataset.

Prosedur Marginal Effect — Dua Langkah Step 1: P₀ = 1 / {1 + exp[−(b₀ + b₁X̄₁ + b₂X̄₂ + ... + bₖX̄ₖ)]} Step 2: P₁ = 1 / {1 + exp[−(b₀ + b₁X̄₁ + ... + bⱼ(X̄ⱼ + δ) + ... + bₖX̄ₖ)]}
Marginal Effect = P₁ − P₀ di mana δ adalah perubahan Xⱼ (biasanya 1 unit atau 0.01 = 1%) Semua X lain tetap di nilai rata-rata
⚠ Marginal Effect Bervariasi per Observasi

Marginal effect 0.20% untuk NPM dalam contoh curriculum berlaku hanya untuk perusahaan dengan karakteristik rata-rata. Untuk perusahaan yang sudah sangat dekat defaultnya (P mendekati 1) atau sangat sehat (P mendekati 0), dampak perubahan NPM yang sama akan jauh lebih kecil karena kurva sigmoid mendatar di ujung-ujung. Di P ≈ 0.5, marginal effect paling besar.

5
Model Fit — Log-Likelihood, Pseudo-R², dan LR Test

Karena logistic regression tidak menggunakan OLS, tidak ada R² konvensional. Tiga metrik pengganti yang perlu dikuasai:

5.1 Log-Likelihood

Log-likelihood (LL) mengukur seberapa baik model menjelaskan data yang diobservasi. Secara teknis: jumlah dari ln(probabilitas prediksi yang benar) untuk semua observasi. Karena probabilitas < 1, ln-nya selalu negatif, sehingga log-likelihood selalu negatif.

🚨 Arah Perbandingan — Selalu Membingungkan

Log-likelihood yang lebih tinggi (mendekati 0) = model lebih baik. Ini berlawanan dengan AIC/BIC (yang lebih rendah = lebih baik). Jangan terkecoh!

Contoh: LL Model A = −198.42, LL Model B = −203.15. Model A lebih baik karena −198.42 > −203.15 (lebih mendekati 0).

Analogi: suhu −5°C "lebih tinggi" daripada −10°C, meski keduanya negatif.

5.2 Pseudo-R²

Pseudo-R² diusulkan sebagai analog R² untuk logit, tapi interpretasinya berbeda:

  • Tidak bisa diinterpretasikan sebagai "% variasi yang dijelaskan" seperti R² OLS
  • Hanya bermakna untuk membandingkan model dengan dependent variable yang sama
  • Nilai Pseudo-R² yang kecil tidak berarti model jelek — dalam logistic regression, nilai 0.05–0.10 saja sudah cukup baik untuk banyak aplikasi
5.3 Likelihood Ratio (LR) Test — Analog Joint F-Test

LR test adalah cara formal untuk menguji apakah sekelompok variabel secara bersama-sama signifikan — analog dengan joint F-test dalam OLS. Membandingkan restricted model vs unrestricted model, tapi menggunakan log-likelihoods bukan SSE.

LR Test Statistic LR = −2 × (LL_restricted − LL_unrestricted) Distribusi chi-square (χ²) dengan df = q (jumlah restrictions) H₀: koefisien variabel yang di-restrict = 0 (restricted model lebih baik) Hₐ: setidaknya satu koefisien ≠ 0 (unrestricted model lebih baik) Tolak H₀ jika LR > χ²_critical(q, α)
💡 Kenapa LR Selalu Positif?

LL_unrestricted selalu ≥ LL_restricted (unrestricted model dengan lebih banyak variabel selalu bisa fit data setidaknya sama baiknya). Keduanya negatif, tapi LL_restricted lebih negatif. Sehingga LL_restricted − LL_unrestricted selalu negatif, dan −2 × (angka negatif) = positif. LR statistic selalu ≥ 0.

Knowledge Check 5.1 — LR Test Lengkap

Model share repurchase (intercept only): LL = −312.68. Full model (CASH, DE, NPM): LL = −304.20. Lakukan LR test pada α = 5% untuk menguji apakah ketiga variabel bersama-sama signifikan. Critical χ²(3) = 7.815.

Step 1: H₀: b_CASH = b_DE = b_NPM = 0 | Hₐ: at least one bⱼ ≠ 0
Step 2: LR statistic, df = 3
Step 3: α = 5%
Step 4: Tolak H₀ jika LR > 7.815
Step 5: LR = −2 × (−312.68 − (−304.20)) = −2 × (−8.48) = 16.96
Step 6: 16.96 > 7.815 → Tolak H₀ → CASH, DE, dan NPM bersama-sama signifikan. Full model lebih baik dari intercept-only model.

Note: p-value dari LR test ini = 0.0007 (ditunjukkan dalam output sebagai LLR-P-Value), sangat signifikan.

6.1 Interpretasi Intercept

Intercept = −0.4738 adalah log-odds repurchase ketika semua X = 0. Probabilitasnya: P = 1/(1 + e^0.4738) = 1/(1 + 1.6063) = 38.37%. Ini adalah probabilitas baseline yang tidak dijelaskan oleh variabel-variabel dalam model.

6.2 Hitung Probabilitas pada Nilai Rata-Rata

Dengan menggunakan mean semua variabel:

Linear predictor = −0.4738 + (−0.9118)(0.083) + (0.9407)(−0.0535) + (−0.3186)(0.9182)
= −0.4738 − 0.0757 − 0.0503 − 0.2926
= −0.8924
P₀ = 1/(1 + e^0.8924) = 1/(1 + 2.441) = 29.06%

Untuk perusahaan dengan karakteristik rata-rata, probabilitas melakukan share repurchase adalah 29.06%.

Ini menunjukkan bahwa setelah mengontrol DE dan NPM, cash ratio tidak memberikan informasi tambahan yang signifikan tentang propensity repurchase. Model bisa disederhanakan dengan drop CASH, tapi LR test overall masih signifikan karena kontribusi DE dan NPM.

Perbandingan Lengkap — OLS vs Logistic Regression
DimensiOLS RegressionLogistic Regression
Dependent variableContinuous (tak terbatas)Binary (0 atau 1)
Model untuk YᵢY = b₀ + b₁X₁ + ... + ε (langsung)ln[P/(1−P)] = b₀ + b₁X₁ + ... (log-odds)
Estimasi koefisienLeast Squares (minimize SSE)MLE (maximize log-likelihood)
Interpretasi bⱼΔY per unit ΔXⱼ (konstan)Δlog-odds per unit ΔXⱼ (bukan ΔP!)
Marginal effectKonstan = bⱼNon-konstan; tergantung nilai semua X
Individual testt-statistic, df = n−k−1z-statistic (same intuition)
Overall model fitF-test (MSR/MSE)LR test (χ², based on log-likelihood)
Goodness of fitR², Adjusted R², AIC, BICPseudo-R², Log-likelihood, LR p-value
Nested model testJoint F-test: (ΔSSE/q) / MSE_unresLR test: −2(LL_res − LL_unres) ~ χ²(q)
Predicted outputPredicted value of Y (continuous)P(Y=1) antara 0 dan 1
✅ Analogi untuk Mengingat

Joint F-test (OLS) ↔ LR test (Logit) — keduanya membandingkan restricted vs unrestricted model

SSE (OLS) ↔ Log-likelihood (Logit) — keduanya mengukur "fit" model terhadap data

R² (OLS) ↔ Pseudo-R² (Logit) — keduanya mengukur goodness of fit, tapi interpretasinya berbeda

t-statistic (OLS) ↔ z-statistic (Logit) — keduanya menguji signifikansi koefisien individual

📝 Latihan Soal — Item Set Vignette
Format CFA L2 asli · 4 soal · ~30 menit
Case Vignette
Arif Wicaksono, CFA, credit analyst di PT Bank Mandiri Tbk, membangun model prediksi default obligasi korporasi Indonesia. Dependent variable: DEFAULT = 1 jika default dalam 12 bulan, 0 jika tidak. n = 400 obligasi, diestimasi dengan MLE.

Model Full (3 variabel):
Intercept: −1.820 (z = −3.12, p = 0.002) | DE: 0.485 (z = 2.84, p = 0.005)
ICR: −0.312 (z = −3.01, p = 0.003) | ROA: −1.640 (z = −2.55, p = 0.011)
LL (full): −198.42 | LL (intercept only): −214.60 | Pseudo-R²: 0.0754

Model Reduced (tanpa ROA): LL = −203.15

Mean values: DE = 1.25, ICR = 3.80, ROA = 0.062
Critical values: z at 5% = ±1.96 | χ²(3) at 5% = 7.815 | χ²(1) at 5% = 3.841
Question 1 of 4
Interpretasi koefisien ICR (−0.312) yang paling tepat adalah:
B adalah jawaban yang benar. Interpretasi standar koefisien logit: bⱼ = perubahan log-odds per unit ΔXⱼ, holding all else constant. Untuk ICR: kenaikan 1 unit ICR menurunkan log-odds default sebesar 0.312. Arah negatif konsisten dengan intuisi — ICR lebih tinggi berarti perusahaan lebih mampu menutup beban bunga → lebih kecil kemungkinan default. Opsi A salah karena menginterpretasikan koefisien seolah OLS. Opsi C secara teknis benar (odds ratio interpretation) tapi bukan cara standar CFA menginterpretasikan koefisien logit — CFA konsisten meminta interpretasi dalam log-odds.
Question 2 of 4
Hitung probabilitas default untuk obligasi dengan nilai rata-rata semua variabel. Linear predictor = −1.820 + 0.485(1.25) + (−0.312)(3.80) + (−1.640)(0.062).
B adalah jawaban yang benar. Hitung linear predictor: −1.820 + 0.485(1.25) − 0.312(3.80) − 1.640(0.062) = −1.820 + 0.606 − 1.186 − 0.102 = −2.502. Kemudian P = 1/(1 + e^2.502) = 1/(1 + 12.20) ≈ 7.6%. Probabilitas default rata-rata sekitar 7.6% — rendah, konsisten dengan bahwa tidak semua obligasi default. Opsi A menggunakan formula yang salah. Opsi C adalah kebetulan numerik — Pseudo-R² bukan probabilitas default.
Question 3 of 4
Apakah ketiga variabel (DE, ICR, ROA) secara bersama-sama signifikan? Gunakan LR test pada α = 5%.
C adalah jawaban yang benar. Formula: LR = −2 × (LL_restricted − LL_unrestricted). Restricted = intercept only (LL = −214.60), Unrestricted = full model (LL = −198.42).
LR = −2 × (−214.60 − (−198.42)) = −2 × (−16.18) = 32.36
32.36 >> 7.815 → tolak H₀ → DE, ICR, ROA bersama-sama berkontribusi signifikan.
Opsi A menggunakan −2 yang hilang (LR pasti negatif kalau formula salah). Opsi B membalik posisi restricted dan unrestricted, menghasilkan nilai negatif yang salah.
Question 4 of 4
Arif ingin menguji apakah ROA memberikan kontribusi inkremental yang signifikan di atas DE dan ICR saja. Model reduced (tanpa ROA) memiliki LL = −203.15. Gunakan LR test pada α = 5%.
B adalah jawaban yang benar. Restricted = model tanpa ROA (LL = −203.15), Unrestricted = full model (LL = −198.42), q = 1.
LR = −2 × (−203.15 − (−198.42)) = −2 × (−4.73) = 9.46
9.46 > χ²(1, α=5%) = 3.841 → tolak H₀ → ROA berkontribusi signifikan.
Opsi A: meski z-statistic individual sudah cukup di sini, LR test adalah cara formal untuk membandingkan nested models — terutama berguna ketika ada beberapa variabel yang di-test bersama (q>1). Opsi C membalik formula dan salah arah — LR selalu positif.
