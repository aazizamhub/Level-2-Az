Theory · 30 min
1
Kenapa Logistic Regression? Masalah Linear Probability Model

Ketika dependent variable bersifat binary (0 atau 1) — misalnya: bankrupt/tidak, repurchase/tidak, winning fund/average fund — kita tidak bisa menggunakan OLS biasa. Kenapa?

Masalah Linear Probability Model (LPM):
Kalau kita pakai OLS langsung: Yᵢ = b₀ + b₁X₁ + b₂X₂ + εᵢ di mana Y = 0 atau 1, predicted value Ŷ bisa menghasilkan angka di atas 1 atau di bawah 0 — tidak valid sebagai probabilitas. Selain itu, hubungan antara probability dan X diasumsikan linear, padahal kenyataannya mendekati S-curve (sigmoidal).

Solusi: Transformasikan P ke dalam log-odds (logit) — yang rangenya dari −∞ sampai +∞ — lalu regresi log-odds terhadap X secara linear. Hasilnya bisa dikonversi balik ke probability yang selalu antara 0 dan 1.

Log-Odds, Odds, dan Probability — Tiga Konsep Terhubung

Odds: Rasio antara probabilitas event terjadi (P) vs tidak terjadi (1−P):

Odds = P / (1 − P) // Kalau P = 0.75 → Odds = 0.75/0.25 = 3 (peluang 3:1)

Log-odds (Logit): Natural log dari odds — ini yang menjadi dependent variable dalam logit model:

Log-odds = ln[P / (1 − P)] = b₀ + b₁X₁ + b₂X₂ + ... + ε // Range: −∞ sampai +∞ → bisa di-regresi secara linear

Balik ke Probability: Dari log-odds model, probability dihitung sebagai:

P = 1 / {1 + exp[−(b₀ + b₁X₁ + b₂X₂ + ...)]} // Selalu antara 0 dan 1 // Kalau linear predictor besar positif → P mendekati 1 // Kalau linear predictor besar negatif → P mendekati 0
⚠ Exam Trap — Interpretasi Koefisien Logit

Koefisien bⱼ dalam logit model bukan perubahan probabilitas per unit X. Yang benar: bⱼ adalah perubahan log-odds per unit kenaikan Xⱼ, holding all else constant. Untuk mendapat perubahan probabilitas yang sebenarnya, kita perlu kalkulasi marginal effect secara terpisah karena hubungannya non-linear.

Interpretasi Intercept: b₀ adalah log-odds dari event ketika semua independent variables = 0. Probabilitasnya = 1 / (1 + e^(−b₀)).

✅ Cara Cepat Estimasi P dari Intercept

Intercept = −0.4738 → P = 1/(1 + e^0.4738) = 1/(1 + 1.606) = 1/2.606 ≈ 0.384 (38.4%). Ini adalah probabilitas event kalau semua X = 0.

3
Marginal Effect & Model Fit — LR Test

Marginal Effect — Prosedur Dua Langkah:

Karena logit non-linear, dampak perubahan Xⱼ sebesar 1 unit terhadap P tergantung pada nilai semua variabel lain (tidak konstan). Cara standar: hitung marginal effect pada rata-rata observasi:

Step 1: Hitung P₀ menggunakan mean semua X dan koefisien model

Step 2: Naikkan Xⱼ sebesar 1 unit (atau increment tertentu), hitung P₁ dengan semua X lain tetap di mean

Marginal Effect = P₁ − P₀

Contoh dari curriculum (share repurchase model):
P₀ (baseline, semua X di mean) = 29.06%
P₁ ketika NPM naik 1% = 29.26%
→ Marginal effect NPM = +0.20% per 1% kenaikan NPM

Penting: marginal effect berbeda untuk setiap observasi! 0.20% ini hanya berlaku untuk perusahaan dengan karakteristik rata-rata.

Tiga Ukuran Model Fit Logit:

MetrikInterpretasiCatatan
Log-likelihoodSelalu negatif. Lebih tinggi (mendekati 0) = lebih baikTidak bermakna sendirian — hanya untuk perbandingan
Pseudo-R²Analog R² tapi interpretasi berbedaHanya valid untuk membandingkan model dengan dependent variable yang sama
LR testAnalog joint F-test di OLS — menguji apakah set variabel berkontribusi signifikanDistribusi chi-square, df = q (jumlah restrictions)
LR = −2 × (LL_restricted − LL_unrestricted) // LL = log-likelihood (angka negatif) // H₀: koefisien variabel yang di-restrict = 0 // Distribusi χ² dengan df = q
⚠ Log-likelihood lebih tinggi = lebih baik

Jangan terkecoh arah perbandingan! −450.4 > −451.66 artinya Model 2 fit lebih baik. Angka yang kurang negatif (lebih mendekati 0) = lebih baik. Ini berlawanan intuisi — biasanya kita mau angka kecil, tapi untuk log-likelihood yang selalu negatif, kita mau yang paling mendekati nol.

Practice — Item Set Vignette
Format CFA L2 · 4 soal · ~30 menit
Case Vignette
Arif Wicaksono, CFA, adalah credit analyst di PT Bank Mandiri Tbk. Ia membangun model untuk memprediksi probabilitas default obligasi korporasi Indonesia (DEFAULT = 1 jika default, 0 jika tidak), menggunakan tiga variabel keuangan: debt-to-equity ratio (DE), interest coverage ratio (ICR), dan return on assets (ROA). Model diestimasi dengan MLE pada 400 observasi.

Hasil logistic regression:
Intercept: −1.820 (z = −3.12, p = 0.002)
DE: 0.485 (z = 2.84, p = 0.005)
ICR: −0.312 (z = −3.01, p = 0.003)
ROA: −1.640 (z = −2.55, p = 0.011)

Log-likelihood (full model): −198.42
Log-likelihood (intercept only): −214.60
Pseudo-R²: 0.0754

Mean values: DE = 1.25, ICR = 3.80, ROA = 0.062

Critical chi-square at 5%, df=3: 7.815. Critical z at 5% (two-tail): ±1.96.
Question 1 of 4
Interpretasi yang paling tepat untuk koefisien DE (0.485) dalam model Arif adalah:
B adalah jawaban yang benar. Koefisien logit mengukur perubahan log-odds, bukan perubahan probabilitas langsung. Karena logistic regression bersifat non-linear, dampak terhadap probabilitas tidak konstan — tergantung pada nilai semua variabel pada saat itu. Opsi A salah karena menginterpretasikan koefisien seolah-olah ini adalah OLS regression. Opsi C secara teknis tidak salah (bisa diinterpretasikan dalam terms of odds ratio e^b), tapi opsi B lebih lengkap dan sesuai dengan cara CFA minta interpretasi koefisien logit — yaitu dalam log-odds.
Question 2 of 4
Hitung probabilitas default untuk perusahaan dengan nilai rata-rata semua variabel (DE=1.25, ICR=3.80, ROA=0.062).
B adalah jawaban yang benar. Hitung linear predictor dulu: −1.820 + 0.485(1.25) + (−0.312)(3.80) + (−1.640)(0.062) = −1.820 + 0.606 − 1.186 − 0.102 = −2.502. P = 1/(1 + e^2.502) = 1/(1 + 12.20) ≈ 7.6%. (Angka exact bergantung pada presisi kalkulasi — dalam konteks vignette, pilih yang paling mendekati.) Opsi A hanya memakai intercept dan mengabaikan semua variabel lain. Opsi C adalah definisi Pseudo-R² yang salah — Pseudo-R² mengukur goodness of fit model, bukan probabilitas.
Question 3 of 4
Untuk menguji apakah ketiga variabel (DE, ICR, ROA) secara bersama-sama signifikan, Arif menggunakan LR test. Hitung LR statistic dan simpulkan pada α = 5%.
C adalah jawaban yang benar. Formula: LR = −2 × (LL_restricted − LL_unrestricted). Restricted model = intercept only (LL = −214.60). Unrestricted = full model (LL = −198.42).
LR = −2 × (−214.60 − (−198.42)) = −2 × (−214.60 + 198.42) = −2 × (−16.18) = 32.36.
32.36 >> 7.815 (χ²_critical df=3, α=5%) → tolak H₀ → DE, ICR, ROA bersama-sama signifikan.
Opsi A salah karena membalik posisi restricted dan unrestricted. Opsi B menggunakan formula yang terbalik dan menghasilkan angka negatif — LR statistic selalu positif karena LL_unrestricted selalu ≥ LL_restricted.
Question 4 of 4
Rekan Arif membandingkan dua spesifikasi model: Model A (full, 3 variabel, LL = −198.42) vs Model B (hanya DE dan ICR, LL = −203.15). Mana yang fit lebih baik, dan bagaimana menentukan apakah ROA signifikan berkontribusi?
C adalah jawaban yang benar. Log-likelihood yang lebih tinggi (mendekati 0) = fit lebih baik. −198.42 > −203.15, jadi Model A fit lebih baik. Untuk menguji kontribusi ROA secara spesifik: Model B adalah restricted (tanpa ROA), Model A adalah unrestricted. LR = −2 × (−203.15 − (−198.42)) = −2 × (−4.73) = 9.46. Dengan df = 1 (satu restriction), critical χ² = 3.841. Karena 9.46 > 3.841 → tolak H₀ → ROA berkontribusi signifikan ke model. Ini adalah analog logit dari joint F-test pada OLS nested models.
