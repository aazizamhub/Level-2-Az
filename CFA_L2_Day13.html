Theory · 30 min
1
Big Data: Karakteristik 4 Vs & Structured vs Unstructured

Big data (a.k.a. alternative data) mencakup data dari financial markets, businesses, governments, individuals, sensors, dan IoT. Yang membedakan big data dari traditional data adalah empat karakteristik — 4 Vs:

VDefinisiImplikasi
VolumeKuantitas data (petabytes, exabytes)Perlu infrastructure komputasi & storage modern
VarietyBeragam sumber: structured, semi-, unstructuredLegal/ethical issues (privacy); butuh integrasi multi-format
VelocityKecepatan data dibuat (real-time)Relevan untuk real-time predictive analytics ("data-in-motion")
VeracityKredibilitas & reliabilitas data (V ke-4 untuk inference)Social media: 10-15% fake; web spam >20%. Critical untuk ML quality

Structured data — tersusun dalam format sistematis (rows = observations, columns = features); langsung dibaca komputer. Contoh: database transaksi, rasio finansial. Unstructured data — ~80% data dunia. Format untuk manusia, bukan komputer. Bentuk: text (tweets, 10-K reports), images (satelit), audio (earnings calls), video. Harus di-transform dulu menjadi structured format sebelum bisa dipakai ML.

Key Insight: Fund managers pakai textual big data (news, social media, filings) untuk ekstrak topics (apa yang dibicarakan) dan sentiment (bagaimana perasaan/pandangan). Studi US menunjukkan positive sentiment di Twitter bisa prediksi trend DJIA sampai 3 hari ke depan dengan akurasi ~87%.

Lima Langkah ML Model Building (Structured vs Text)

Untuk structured data, ML model building ada 5 langkah iteratif:

1. Conceptualization → 2. Data Collection → 3. Data Preparation & Wrangling
→ 4. Data Exploration → 5. Model Training
// Iteratif — insights dari satu iterasi inform iterasi berikutnya.

Untuk unstructured text data, 4 langkah pertama berbeda (tujuan akhirnya menghasilkan structured output):

StepStructured DataText Data
1ConceptualizationText Problem Formulation
2Data CollectionData (Text) Curation
3Data Prep & WranglingText Prep & Wrangling
4Data ExplorationText Exploration
5Model TrainingModel Training (sama)
Key Insight: Output dari text ML model (misal sentiment scores) bisa di-combine dengan structured features lain atau dipakai langsung untuk forecasting/analysis. Step Data/Text Preparation & Wrangling biasanya mengkonsumsi paling banyak waktu project (sering disebut "80% of the work").
3
Structured Data: Cleansing (6 Errors) & Preprocessing

Data Cleansing — identifikasi & perbaiki errors. Ada 6 jenis error yang harus diingat:

ErrorDefinisiContoh
IncompletenessData hilang (NA)Kolom "Salary" kosong
InvalidityDi luar range masuk akalDate of birth = 1/1/1900; income negatif
InaccuracyBukan nilai sebenarnyaCredit card = "Don't know"
InconsistencyBertentangan dengan field lainName "Ms. XYZ" tapi Gender = M
Non-uniformityFormat tidak identikTanggal "12/5/1970" vs "15 Jan 1975"
DuplicationObservasi duplikatRow 7 = Row 3 (customer sama)

Data Preprocessing (Wrangling) — 5 transformasi utama untuk menyiapkan data training:

1. Extraction: buat variabel baru dari yg ada (DoB → Age)
2. Aggregation: gabungkan variabel (Salary + Other Income → Total Income)
3. Filtration: hapus rows yang tidak relevan (customer non-US)
4. Selection: hapus columns yang tidak perlu (Name, jika ada ID)
5. Conversion: sesuaikan tipe variabel (nominal, ordinal, continuous, categorical)

Outliers: deteksi via standard deviation (±3σ) atau IQR (di luar Q3+1.5·IQR atau Q1−1.5·IQR). Penanganan: trimming (hapus top/bottom %) atau winsorization (replace dengan nilai batas non-outlier).

// Normalization (min-max scaling): range [0,1]
X_norm = (X − X_min) / (X_max − X_min)

// Standardization (z-score): mean = 0, std = 1
X_std = (X − μ) / σ
// Normalization sensitif outliers — bersihkan outlier dulu.
// Standardization asumsi data normally distributed.
⚠ Exam Trap

Normalization ≠ Standardization. Normalization (min-max) mengharuskan data bebas outlier; range-nya selalu [0,1]. Standardization (z-score) asumsinya data normally distributed; hasilnya bisa negatif/positif. Keduanya penting untuk ML methods seperti SVM dan ANN yang sensitif terhadap skala.

4
Text Processing: Cleansing → Tokenization → Normalization → BOW → DTM

Text Cleansing (4 operasi, urutan penting):

1. Remove HTML tags (regex: "<.*?>")
2. Remove/replace punctuations (%, $, ? → /percentSign/, /dollarSign/, /questionMark/)
3. Remove/replace numbers (→ /number/)
4. Remove extra white spaces
// Urutan penting: jika punctuation dihapus sebelum numbers, "1.2 million" → "12 million"

Text Wrangling (Preprocessing) — normalisasi token:

  1. Tokenization: pecah teks jadi tokens (biasanya word-level)
  2. Lowercasing: hilangkan perbedaan case ("The" = "the")
  3. Stop words removal: buang "the", "is", "a" — tidak bermakna untuk ML
  4. Stemming: rule-based → stem ("analyzed" → "analyz"). Porter's algorithm.
  5. Lemmatization: algorithmic, morphological root ("analyzed" → "analyze"). Lebih akurat, lebih mahal.

Output akhir: Bag-of-Words (BOW) — kumpulan distinct tokens. Lalu bangun Document Term Matrix (DTM): rows = documents, columns = tokens, cells = frequency counts. DTM adalah structured data yang siap di-training ML.

// N-grams: sequences of n words
unigram (n=1): "market"
bigram (n=2): "market_today"
trigram (n=3): "went_to_market"
// N-grams menyelamatkan informasi posisi/konteks yang hilang di BOW biasa.
⚠ Exam Trap

BOW tidak memelihara urutan kata — "no market" dan "market no" sama di unigram BOW. Gunakan n-grams untuk preserve context (misalnya bigram "no_market" menangkap negasi). Juga: stop word removal bisa menghilangkan informasi penting — misalnya "no" yang merupakan negasi sering jadi stop word. Karena itu custom stop word list sering dibutuhkan sesuai domain.

Mini Case: Data Science Team PT Indo Premier
4 soal · Vignette-based · Pilih jawaban terbaik
Vignette
Rina Wulandari, head of data science di PT Indo Premier Sekuritas, sedang membangun platform analytics untuk memantau 300 saham IDX. Timnya mengumpulkan data dari berbagai sumber: database internal (harga saham, volume, rasio finansial), feed dari Bursa Efek Indonesia (BEI), dan 12.000 transkrip earnings calls perusahaan IDX dalam 3 tahun terakhir. Tim menemukan masalah berikut saat review data:

Issue 1: Pada dataset investor profile, kolom "Tanggal Lahir" mencatat tanggal "1/1/1900" untuk 47 nasabah; kolom "Pendapatan Tahunan" kosong untuk 82 nasabah; dan 15 baris memiliki nasabah yang sama tercatat dua kali.

Issue 2: Salah satu analis junior, Bima Saputra, baru saja menjalankan SVM (support vector machine) langsung pada dataset yang memiliki dua features: "Total Aset" (range Rp 1 juta – Rp 50 miliar) dan "Usia Nasabah" (range 18–85). Model memberikan prediksi aneh di mana "Total Aset" seolah mendominasi semua keputusan.

Issue 3: Dalam pipeline NLP untuk transkrip earnings calls, tim menghapus punctuation sebelum menangani numbers. Hasil: kalimat "Revenue grew 12.5% to USD 1.2 million" menjadi "Revenue grew 125 to USD 12 million" yang salah interpretasi.

Issue 4: Tim ingin menangkap negasi dalam teks (misalnya "no growth" vs "strong growth"). Pipeline saat ini menggunakan unigram BOW dan menghilangkan stop words termasuk kata "no" dan "not".
Soal 1 / 4
Berdasarkan Issue 1, jenis error yang ada pada data investor profile adalah:
C adalah jawaban yang benar. Tanggal lahir 1/1/1900 = invalidity error (di luar range masuk akal untuk umur manusia modern). Pendapatan kosong = incompleteness error. Nasabah tercatat dua kali = duplication error. A salah karena melewatkan duplication dan incompleteness. B salah karena tidak ada indikasi non-uniformity di data yang disebutkan (format tidak seragam).
Soal 2 / 4
Untuk memperbaiki masalah Bima di Issue 2 (SVM dengan features berbeda skala drastis), tindakan yang paling tepat adalah:
B adalah jawaban yang benar. SVM (dan ANN) sangat sensitif terhadap skala features. Features dengan range jauh lebih besar ("Total Aset") akan mendominasi model. Solusinya: scaling — normalization (min-max ke [0,1]) atau standardization (z-score, mean=0, std=1) — agar semua features berkontribusi setara. A salah karena SVM bisa sangat efektif untuk data finansial setelah scaling dilakukan. C salah karena menghapus feature tanpa justifikasi statistis adalah pemborosan informasi.
Soal 3 / 4
Untuk menghindari masalah seperti Issue 3 di pipeline NLP, urutan operasi text cleansing yang benar adalah:
A adalah jawaban yang benar. Urutan cleansing matters: HTML tags dulu (karena bisa mengandung punctuation/numbers), lalu punctuations (dengan annotation seperti /dollarSign/, /percentSign/ untuk simbol penting), kemudian numbers (dengan /number/ annotation), terakhir white spaces (yang muncul setelah operasi sebelumnya). Dalam Issue 3, numbers harus ditangani sebelum punctuations — atau punctuation harus diganti dengan annotation, bukan dihapus — supaya "1.2 million" tidak menjadi "12 million". B salah (urutan terbalik). C salah karena itu adalah tahap wrangling/normalization, bukan cleansing.
Soal 4 / 4
Solusi terbaik untuk Issue 4 (menangkap negasi dalam teks) adalah:
B adalah jawaban yang benar. BOW unigram kehilangan konteks karena tidak menangkap urutan kata dan menghapus negasi ("no", "not") sebagai stop words. Dua solusi: (1) gunakan bigrams/trigrams sehingga "no_growth" menjadi token tunggal yang merepresentasikan negasi, dan/atau (2) custom stop word list yang mempertahankan kata-kata negasi penting. A salah karena menambah data tidak memperbaiki limitasi struktural BOW terhadap negasi. C salah karena linear regression bukan pilihan untuk binary classification sentiment (positive/negative); juga tidak mengatasi masalah representasi negasi.
