1
Masalah Linear Probability Model & Motivasi Logistic Regression

Dalam analisis investasi, kita sering menghadapi pertanyaan yang jawabannya bukan angka kontinu melainkan kategori biner: apakah perusahaan akan bangkrut atau tidak? Apakah nasabah akan default atau tidak? Apakah sentimen positif atau negatif? Variabel dependen seperti ini disebut qualitative dependent variable (categorical dependent variable), yang mengambil nilai diskrit — biasanya 1 (event terjadi) dan 0 (tidak terjadi).

Linear Probability Model (LPM) — Pendekatan Naif

Pendekatan paling sederhana adalah langsung meregresikan variabel binary (Y = 0/1) terhadap variabel independen menggunakan OLS biasa. Model ini disebut Linear Probability Model (LPM):

Linear Probability Model Yi = b₀ + b₁X1i + b₂X2i + b₃X3i + εi Dimana Y = 1 jika event terjadi (misal bangkrut), Y = 0 jika tidak

LPM memiliki dua masalah fundamental yang serius:

  • Predicted values di luar rentang [0, 1]: Karena model linear, nilai prediksi Ŷ bisa lebih besar dari 1 atau kurang dari 0 tergantung pada koefisien dan nilai variabel independen. Ini invalid karena probabilitas secara definisi harus berada di antara 0 dan 1.
  • Asumsi linearitas yang unrealistic: LPM mengasumsikan hubungan antara setiap variabel independen dan probabilitas event bersifat linear di seluruh rentang. Pada kenyataannya, dampak perubahan variabel (misalnya debt-to-equity) terhadap probabilitas bangkrut tidak konstan — di level sangat rendah atau sangat tinggi, dampaknya biasanya lebih kecil dibanding di level moderat.
Contoh Konkret

Bayangkan model: P(bankrupt) = 0.30 + 0.40 × DE. Untuk perusahaan dengan DE = 2.0, prediksi = 0.30 + 0.40(2.0) = 1.10 — probabilitas 110%? Tidak masuk akal. Sebaliknya, untuk DE = −1.0, prediksi = −0.10 — probabilitas negatif?

Untuk mengatasi kedua masalah ini, kita memerlukan transformasi non-linear yang memetakan output model ke rentang (0, 1). Transformasi yang paling umum digunakan dalam statistik dan machine learning adalah logistic transformation, yang menghasilkan model logistic regression (atau model logit).

Knowledge Check 1
Seorang analis membangun model untuk memprediksi apakah perusahaan akan melakukan stock split (Y = 1) atau tidak (Y = 0). Ia menggunakan OLS dan mendapatkan Ŷ = 1.15 untuk salah satu perusahaan. Apa masalah utama dari hasil ini, dan apa solusinya?
Masalah: Predicted probability 1.15 (atau 115%) berada di luar rentang valid [0, 1]. Ini terjadi karena OLS linear tidak membatasi output — ini kelemahan fundamental Linear Probability Model. Solusi: Gunakan logistic regression yang mentransformasi output melalui fungsi sigmoid, sehingga predicted probability selalu berada di antara 0 dan 1.

Formulasi Model: Log-Odds, Odds Ratio & Fungsi Sigmoid
Odds dan Log-Odds

Sebelum memahami logistic regression, kita perlu memahami konsep odds. Jika P adalah probabilitas suatu event terjadi, maka:

Odds Odds = P / (1 − P) Rasio probabilitas event terjadi terhadap probabilitas event tidak terjadi

Contoh: jika probabilitas perusahaan bangkrut P = 0.75, maka odds = 0.75 / (1 − 0.75) = 0.75 / 0.25 = 3. Artinya odds bangkrut adalah 3 banding 1 — probabilitas bangkrut tiga kali lebih besar daripada probabilitas tidak bangkrut.

Karena odds bisa bernilai dari 0 sampai +∞, kita ambil logaritma natural-nya untuk mendapatkan log-odds (juga dikenal sebagai logit), yang bisa bernilai dari −∞ sampai +∞. Ini yang menjadi variabel dependen dalam logistic regression:

Logistic Regression Model (Logit Model) ln(P / (1 − P)) = b₀ + b₁X₁ + b₂X₂ + ... + bₖXₖ + ε Log-odds diregresikan secara linear terhadap variabel independen
Fungsi Sigmoid — Dari Log-Odds ke Probabilitas

Setelah mengestimasi koefisien model, kita perlu mengkonversi log-odds kembali ke probabilitas. Ini dilakukan dengan menerapkan inverse logit, yang menghasilkan fungsi sigmoid (berbentuk S-curve):

Sigmoid Function (Inverse Logit) P = 1 / (1 + e−(b₀ + b₁X₁ + b₂X₂ + ... + bₖXₖ)) Selalu menghasilkan nilai antara 0 dan 1 — valid sebagai probabilitas

Kita definisikan z = b₀ + b₁X₁ + ... + bₖXₖ. Maka P = 1/(1 + e−z). Perhatikan sifat-sifat sigmoid:

Estimasi: Maximum Likelihood Estimation (MLE)

Berbeda dengan linear regression yang diestimasi menggunakan Ordinary Least Squares (OLS), logistic regression diestimasi menggunakan Maximum Likelihood Estimation (MLE). MLE mencari set koefisien (b₀, b₁, ..., bₖ) yang membuat data observasi paling mungkin terjadi — yaitu yang memaksimalkan likelihood function.

Karena setiap observasi bersifat biner (0 atau 1), distribusi probabilitas yang digunakan untuk membangun likelihood function adalah distribusi binomial. MLE membutuhkan iterasi numerik (tidak ada closed-form solution seperti OLS), sehingga selalu dihitung menggunakan software statistik.

Logistic regression mengasumsikan error term mengikuti distribusi logistik, yang bentuknya mirip distribusi normal tetapi dengan ekor yang lebih tebal (fatter tails).

Perbandingan OLS vs MLE

OLS: Meminimalkan jumlah kuadrat residual (Σε²). Cocok untuk dependent variable kontinu. Menghasilkan R² sebagai goodness of fit.

MLE: Memaksimalkan probabilitas data terjadi. Cocok untuk dependent variable biner/categorical. Menghasilkan log-likelihood dan pseudo-R² sebagai ukuran fit.

Knowledge Check 2
Jika probabilitas suatu perusahaan melakukan share repurchase adalah P = 0.20, berapakah: (a) odds repurchase, (b) log-odds, dan (c) berapa nilai z pada sigmoid function yang menghasilkan P = 0.20?
(a) Odds = P/(1−P) = 0.20/0.80 = 0.25. Artinya probabilitas repurchase hanya seperempat dari probabilitas tidak repurchase. (b) Log-odds = ln(0.25) = −1.386. Nilai negatif menunjukkan event kurang likely dari non-event. (c) z = −1.386. Verifikasi: P = 1/(1 + e1.386) = 1/(1 + 3.999) = 1/4.999 ≈ 0.20. ✓

Interpretasi Koefisien & Marginal Effects
Interpretasi Koefisien sebagai Perubahan Log-Odds

Koefisien dalam logistic regression tidak bisa diinterpretasikan sama seperti koefisien OLS. Dalam OLS, b₁ berarti "perubahan Y per unit perubahan X₁." Dalam logistic regression, b₁ berarti perubahan log-odds per unit perubahan X₁, holding all other variables constant. Ini bukan perubahan probabilitas!

Hypothesis test untuk koefisien logistic regression mirip dengan OLS. Software biasanya menghasilkan z-statistic (bukan t-statistic) karena MLE menggunakan distribusi normal asymptotic. Null hypothesis: H₀: bⱼ = 0 (variabel tidak signifikan).

Odds Ratio — Interpretasi yang Lebih Intuitif

Karena "perubahan log-odds" sulit dibayangkan, interpretasi yang lebih praktis menggunakan odds ratio:

Odds Ratio Odds Ratio = ebⱼ Faktor pengali odds untuk setiap kenaikan 1 unit Xⱼ, ceteris paribus
Exhibit 2 — Contoh Interpretasi Odds Ratio
VariabelKoefisien (bⱼ)Odds Ratio (ebⱼ)Interpretasi
DE (Debt/Equity)+0.802.226↑1 unit DE → odds bangkrut ×2.23
ROE−1.500.223↑1 unit ROE → odds bangkrut ×0.22 (↓78%)
SIZE (ln aset)−0.300.741↑1 unit SIZE → odds bangkrut ×0.74 (↓26%)

Jika odds ratio > 1, kenaikan variabel meningkatkan odds event. Jika odds ratio < 1, kenaikan variabel menurunkan odds event. Odds ratio = 1 berarti variabel tidak berpengaruh.

Marginal Effects — Dampak pada Probabilitas

Pertanyaan yang paling sering diajukan praktisi: "Kalau DE naik 1%, berapa perubahan probabilitas bangkrut?" Ini membutuhkan perhitungan marginal effect, yang dalam logistic regression bersifat non-konstan — dampaknya berbeda-beda tergantung level semua variabel.

Cara menghitung marginal effect untuk "perusahaan rata-rata":

  • Hitung P₀ menggunakan nilai mean semua variabel independen: P₀ = 1/(1 + e−z₀)
  • Naikkan variabel yang ingin diteliti sebesar 1 unit (atau 1%), hitung P₁ = 1/(1 + e−z₁)
  • Marginal effect = P₁ − P₀
Perbedaan Kunci: OLS vs Logistic

Dalam OLS, marginal effect = b₁ (konstan di semua level X). Dalam logistic regression, marginal effect berubah-ubah. Di sekitar P = 0.5 (inflection point sigmoid), perubahan 1 unit X punya dampak terbesar pada probabilitas. Di ekstrem (P mendekati 0 atau 1), dampaknya kecil karena kurva sigmoid sangat landai.

Exam Trap

Jangan pernah menginterpretasikan koefisien logistic regression sebagai perubahan probabilitas langsung. Jika b₁ = 2.50, ini bukan berarti "probabilitas naik 2.50 per unit X." Ini berarti log-odds naik 2.50, atau odds menjadi e2.50 = 12.18 kali lipat. Perubahan probabilitas harus dihitung terpisah menggunakan marginal effects.

Knowledge Check 3
Model logistic regression untuk default: ln(P/(1−P)) = −2.0 + 3.0×DE. Untuk perusahaan dengan DE = 1.0, berapa probabilitas default? Jika DE naik menjadi 1.5, berapa probabilitas baru dan marginal effect dari kenaikan 0.5 unit DE?
DE = 1.0: z = −2.0 + 3.0(1.0) = 1.0. P = 1/(1 + e−1) = 1/1.368 = 73.1%.
DE = 1.5: z = −2.0 + 3.0(1.5) = 2.5. P = 1/(1 + e−2.5) = 1/1.082 = 92.4%.
Marginal effect: 92.4% − 73.1% = +19.3 percentage points untuk kenaikan 0.5 unit DE.
Perhatikan bahwa perubahan 0.5 unit DE dari 1.0 ke 1.5 memberikan dampak +19.3pp. Jika kita hitung dari DE=2.0 ke 2.5: z₁=4.0→P=98.2%, z₂=5.5→P=99.6%, marginal effect hanya +1.4pp. Ini menunjukkan sifat non-linear dari marginal effects.

Evaluasi Model: Likelihood Ratio Test, Pseudo-R², Confusion Matrix & ROC/AUC
Likelihood Ratio (LR) Test

Karena logistic regression tidak diestimasi menggunakan OLS, kita tidak bisa menggunakan F-test tradisional untuk menguji signifikansi keseluruhan model. Sebagai gantinya, kita menggunakan Likelihood Ratio (LR) test, yang membandingkan fit antara model unrestricted (lengkap) dan model restricted (biasanya intercept-only model).

Likelihood Ratio Test Statistic LR = −2 × (Log-likelihoodrestricted − Log-likelihoodunrestricted) Terdistribusi chi-squared dengan q degrees of freedom (q = jumlah restriksi/variabel yang diuji)

Log-likelihood selalu bernilai negatif (semakin mendekati 0, semakin baik fit). H₀: model restricted (lebih kecil) lebih baik. Jika LR > critical chi-squared value, tolak H₀ — artinya model unrestricted secara signifikan lebih baik.

Exhibit 4 — Contoh LR Test
ModelLog-Likelihood
Restricted (intercept only)−312.68
Unrestricted (3 variabel)−304.20

LR = −2 × (−312.68 − (−304.20)) = −2 × (−8.48) = 16.96. Critical χ² (df=3, α=5%) = 7.815. Karena 16.96 > 7.815 → tolak H₀. Model unrestricted lebih baik.

Pseudo-R²

Logistic regression tidak memiliki R² yang setara dengan OLS karena model tidak diestimasi menggunakan least squares. Sebagai alternatif, software menghasilkan Pseudo-R² yang mengukur explained variation. Namun, pseudo-R² harus diinterpretasikan dengan hati-hati: hanya valid untuk membandingkan spesifikasi berbeda dari model yang sama (dataset sama), bukan untuk membandingkan model dari dataset yang berbeda.

Confusion Matrix

Untuk mengevaluasi seberapa baik model mengklasifikasikan observasi, kita menggunakan confusion matrix. Setelah memilih threshold (biasanya 0.5), setiap observasi diklasifikasikan sebagai predicted positive (P ≥ threshold) atau predicted negative (P < threshold), lalu dibandingkan dengan nilai aktualnya.

Exhibit 5 — Struktur Confusion Matrix
Predicted = 1 (Positive)Predicted = 0 (Negative)
Actual = 1True Positive (TP)False Negative (FN) — Type II
Actual = 0False Positive (FP) — Type ITrue Negative (TN)

Dari confusion matrix, kita menghitung beberapa metrik kinerja:

Classification Metrics Accuracy = (TP + TN) / (TP + TN + FP + FN) Precision = TP / (TP + FP) → dari predicted positive, berapa yang benar? Recall = TP / (TP + FN) → dari actual positive, berapa yang terdeteksi? Specificity = TN / (TN + FP) → dari actual negative, berapa yang benar? F1 Score = 2 × (Precision × Recall) / (Precision + Recall)
Precision vs Recall — Trade-off

Dalam konteks credit risk, recall sering lebih penting: kita ingin mendeteksi semua perusahaan yang benar-benar akan default (minimize FN), bahkan kalau ada beberapa false alarm (FP lebih banyak). Menurunkan threshold dari 0.5 ke 0.3 meningkatkan recall tapi menurunkan precision. Sebaliknya, dalam fraud detection dengan volume sangat tinggi, precision mungkin lebih penting untuk mengurangi investigasi yang sia-sia.

ROC Curve & AUC

ROC (Receiver Operating Characteristic) curve adalah grafik yang memplot True Positive Rate (recall/sensitivity) terhadap False Positive Rate (1 − specificity) di berbagai threshold. Kurva ini memberikan gambaran menyeluruh tentang performa model di semua threshold — bukan hanya di satu threshold tertentu.

AUC (Area Under the ROC Curve) merangkum performa keseluruhan model dalam satu angka:

Exhibit 6 — Interpretasi AUC
AUC RangeInterpretasi
0.50Sama dengan tebak random — model tidak berguna
0.50 – 0.70Poor discrimination
0.70 – 0.80Acceptable discrimination
0.80 – 0.90Good/excellent discrimination
0.90 – 1.00Outstanding (jarang tercapai di finance)
Knowledge Check 4
Sebuah model distress menghasilkan confusion matrix: TP=20, FP=10, FN=5, TN=65 dari 100 observasi. Hitung accuracy, precision, dan recall. Mana metrik yang paling relevan untuk bank yang ingin meminimalkan unexpected loan losses?
Accuracy = (20+65)/100 = 85%. Precision = 20/(20+10) = 66.7%. Recall = 20/(20+5) = 80.0%.
Untuk bank yang ingin meminimalkan unexpected loan losses, recall adalah yang paling relevan. Recall mengukur persentase perusahaan yang benar-benar distress dan berhasil terdeteksi model. FN (5 perusahaan distress yang tidak terdeteksi) menyebabkan unexpected losses — ini yang ingin diminimalkan. Beberapa false positives (FP=10) lebih bisa ditoleransi karena bank hanya perlu melakukan due diligence tambahan.

Aplikasi: Logistic Regression dalam Machine Learning & Finance

Logistic regression adalah salah satu algoritma supervised learning yang paling fundamental dalam machine learning. Di CFA Level II, logistic regression menjadi jembatan antara materi "traditional statistics" (LM1–LM4) dan materi machine learning (LM5+).

Aplikasi Utama di Finance
  • Credit Scoring & Default Prediction: Bank dan lembaga rating menggunakan logistic regression untuk mengestimasi probabilitas default berdasarkan rasio keuangan perusahaan (debt-to-equity, interest coverage, ROA, dll). Model Altman Z-score yang lebih modern sering menggunakan logit framework.
  • Sentiment Classification: Logistic regression dikombinasikan dengan Natural Language Processing (NLP) untuk mengklasifikasikan sentimen teks keuangan (press releases, earnings calls, annual reports) menjadi positif (1) atau negatif (0). Variabel independen berupa tokens — kata kunci atau frasa dari teks keuangan.
  • Fraud Detection: Mengklasifikasikan transaksi sebagai fraudulent (1) atau legitimate (0) berdasarkan pola transaksi (jumlah, frekuensi, lokasi, dll).
  • M&A Prediction: Memprediksi apakah perusahaan akan menjadi target akuisisi berdasarkan karakteristik fundamental (undervaluation, cash holdings, market share).
Koneksi ke Neural Networks

Fungsi sigmoid yang digunakan dalam logistic regression adalah activation function yang sama yang digunakan di output layer neural networks untuk binary classification. Sebuah neural network tanpa hidden layer yang menggunakan sigmoid activation pada dasarnya adalah logistic regression. Oleh karena itu, memahami logistic regression adalah fondasi untuk memahami deep learning.

Konteks Indonesia

Di pasar modal Indonesia, OJK (Otoritas Jasa Keuangan) menggunakan model-model berbasis logistic regression untuk early warning system risiko perbankan. Bank-bank besar seperti BCA, BRI, dan Mandiri juga menggunakan logit models dalam internal credit scoring mereka. Model ini mengestimasi probability of default (PD) yang menjadi input kunci dalam perhitungan modal minimum sesuai Basel framework.

Knowledge Check 5
Seorang analis NLP membangun model logistic regression untuk sentiment classification pada press releases perusahaan IDX. Variabel dependen: sentiment (1=positif, 0=negatif). Variabel independen: kata-kata seperti "growth", "profit", "loss", "restructuring". Kenapa logistic regression cocok di sini, dan apa hubungannya dengan neural networks?
Logistic regression cocok karena dependent variable bersifat biner (positif/negatif), bukan kontinu. Model mengestimasi probabilitas teks memiliki sentimen positif berdasarkan keberadaan dan frekuensi kata-kata tertentu. Hubungan dengan neural networks: Logistic regression adalah neural network paling sederhana — single layer tanpa hidden layers, menggunakan sigmoid activation function. Neural network yang lebih kompleks (dengan hidden layers) bisa menangkap pola non-linear antar tokens yang logistic regression tidak bisa, tetapi prinsip dasar outputnya sama: sigmoid function menghasilkan probabilitas antara 0 dan 1.

Case Vignette

Question 1 of 6
Pada level signifikansi 5%, variabel mana yang TIDAK signifikan dalam model Siti?
B adalah jawaban yang benar. VOLATILITY memiliki p-value = 0.096, yang lebih besar dari α = 0.05, sehingga kita gagal menolak H₀: b(VOLATILITY) = 0 pada level 5%. Variabel ini tidak signifikan secara statistik. Opsi A salah — koefisien DE positif justru sesuai teori (DE tinggi → risiko default tinggi). Opsi C salah — dummy variable bisa dan harus diuji signifikansinya; SIZE_DUMMY signifikan (p = 0.0102 < 0.05).
Question 2 of 6
Odds ratio untuk variabel ICR adalah closest to:
A adalah jawaban yang benar. Odds ratio = eb = e−2.40 = 0.0907 ≈ 0.091. Karena odds ratio < 1, kenaikan 1 unit ICR menurunkan odds default. Secara spesifik, odds menjadi 9.1% dari sebelumnya — penurunan ~91%. Ini masuk akal: perusahaan dengan interest coverage lebih tinggi lebih mampu membayar bunga. Opsi B salah — 2.40 adalah koefisien (log-odds), bukan odds ratio. Opsi C menghitung e+2.40 = 11.02, yang salah tanda (seharusnya negatif).
Question 3 of 6
Probabilitas default untuk PT Indo Textile (DE=1.20, VOL=0.45, ICR=0.80, small cap) adalah closest to:
B adalah jawaban yang benar. Step 1 — hitung z: z = −3.200 + 1.80(1.20) + 0.50(0.45) − 2.40(0.80) + 0.90(1) = −3.200 + 2.160 + 0.225 − 1.920 + 0.900 = −1.835. Step 2 — sigmoid: P = 1/(1 + e1.835) = 1/(1 + 6.264) = 1/7.264 ≈ 13.8%. Meskipun PT Indo Textile memiliki DE tinggi dan small cap, interest coverage ratio 0.80 dan efek intercept yang besar (−3.20) menekan probabilitas. Opsi A (5.2%) sesuai z ≈ −2.9, terlalu rendah. Opsi C (49%) sesuai z ≈ 0, terlalu tinggi.
Question 4 of 6
Recall (sensitivity) model Siti berdasarkan confusion matrix out-of-sample adalah:
B adalah jawaban yang benar. Recall = TP/(TP+FN) = 18/(18+4) = 18/22 = 81.8%. Recall mengukur: dari semua perusahaan yang benar-benar default (22), berapa persen yang berhasil dideteksi model (18). Opsi A menghitung precision = TP/(TP+FP) = 18/25 = 72.0%. Opsi C menghitung accuracy = (TP+TN)/Total = (18+71)/100 = 89.0%.
Question 5 of 6
LR test statistic untuk model Siti, dibandingkan dengan critical χ² (df=4, α=5%) = 9.488, menghasilkan kesimpulan:
A adalah jawaban yang benar. LR = −2 × (LLR − LLU) = −2 × (−168.30 − (−142.50)) = −2 × (−25.80) = 51.60. Karena 51.60 >> 9.488 (critical value), kita menolak H₀ dengan sangat kuat. Model unrestricted dengan 4 variabel secara signifikan lebih baik daripada intercept-only model. Opsi B salah — 25.80 adalah selisih log-likelihood, bukan LR statistic (lupa dikali −2). Opsi C salah perhitungannya.

Siti ingin membandingkan pseudo-R² modelnya (0.153) dengan pseudo-R² model lain yang diestimasi menggunakan dataset dari pasar saham Thailand. Pernyataan mana yang paling tepat?
C adalah jawaban yang benar. Sesuai kurikulum CFA, pseudo-R² harus diinterpretasikan dengan hati-hati dan hanya bisa digunakan untuk membandingkan spesifikasi berbeda dari model yang sama (artinya dataset yang sama). Membandingkan pseudo-R² antar dataset (Indonesia vs Thailand) tidak valid. Opsi A salah karena mengasumsikan perbandingan cross-dataset valid. Opsi B salah karena menginterpretasikan pseudo-R² seolah-olah identik dengan R² dari OLS — pseudo-R² tidak bisa diinterpretasikan langsung sebagai "proporsi variasi yang dijelaskan."
