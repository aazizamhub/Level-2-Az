1
Kenapa Butuh Logistic Regression?

Kalau dependent variable lo bukan angka kontinu tapi binary outcome (0 atau 1) — misalnya apakah perusahaan bangkrut atau tidak, apakah saham naik atau turun — linear regression punya masalah serius.

Masalah Linear Probability Model (LPM): Kalau lo paksa pakai OLS dengan Y = 0/1, predicted values bisa kurang dari 0 atau lebih dari 1. Ini nggak make sense karena probabilitas harus di antara 0 dan 1. Selain itu, LPM mengasumsikan hubungan linear antara X dan probabilitas, yang sering unrealistic — misal, dampak debt-to-equity terhadap probabilitas bangkrut pasti nggak linear di level sangat rendah/tinggi.

Linear Probability Model: Yi = b₀ + b₁X₁ + b₂X₂ + ε
// Problem: Ŷ bisa < 0 atau > 1 → invalid sebagai probabilitas!

Solusinya: kita transformasi probabilitas menggunakan logistic function sehingga output selalu terbatas antara 0 dan 1, membentuk kurva sigmoid (S-shape).

⚠ Exam Trap

Jangan samakan logistic regression dengan linear regression. Logistic regression bukan diestimasi pakai OLS (least squares) tapi pakai Maximum Likelihood Estimation (MLE). MLE mencari koefisien yang paling "likely" menghasilkan data yang kita observasi.

2
Log-Odds, Odds Ratio & Fungsi Sigmoid

Logistic regression bekerja dengan mentransformasi probabilitas P menjadi log-odds (logit), lalu meregresikan log-odds tersebut terhadap variabel independen secara linear.

Logit: ln(P / (1 − P)) = b₀ + b₁X₁ + b₂X₂ + ... + bₖXₖ
// P/(1−P) = odds; ln(odds) = log-odds = logit

Odds: Rasio P terhadap (1 − P). Misal P(bankrupt) = 0.75, maka odds = 0.75/0.25 = 3 → "3 banding 1 akan bangkrut."

Odds Ratio: Kalau koefisien b₁ = 0.5, maka kenaikan 1 unit X₁ meningkatkan odds sebesar faktor e0.5 = 1.649 → odds naik 64.9%, holding other variables constant.

Odds Ratio = eb₁
// Kenaikan 1 unit X₁ → odds dikali e^(b₁)

Dari log-odds ke probabilitas (sigmoid):

P = 1 / (1 + e−(b₀ + b₁X₁ + ... + bₖXₖ))
// Fungsi sigmoid: selalu menghasilkan nilai antara 0 dan 1
Key Insight: Karena sigmoid non-linear, dampak perubahan 1 unit X terhadap probabilitas tergantung pada level semua variabel lainnya. Di sekitar P = 0.5 (inflection point), perubahan X punya dampak paling besar terhadap P. Di ujung-ujung (P mendekati 0 atau 1), dampaknya kecil. Ini beda banget dari OLS yang punya marginal effect konstan.
3
Interpretasi Koefisien & Marginal Effects

Koefisien dalam logistic regression nggak bisa langsung diinterpretasikan sebagai "perubahan probabilitas per unit X" (beda dari OLS). Yang benar:

InterpretasiPenjelasan
b₁ (raw) Perubahan log-odds per kenaikan 1 unit X₁, ceteris paribus
eb₁ (odds ratio) Faktor pengali odds per kenaikan 1 unit X₁. Jika > 1: odds naik; jika < 1: odds turun
Marginal Effect Perubahan probabilitas per kenaikan 1 unit X₁ — harus dihitung di titik tertentu (biasanya mean)

Cara hitung marginal effect: (1) Hitung P₀ menggunakan mean semua X. (2) Naikkan X₁ sebesar 1 unit, hitung P₁. (3) Marginal effect = P₁ − P₀. Ini tergantung lokasi — bukan angka tunggal!

⚠ Exam Trap

Jangan bilang "kenaikan 1 unit X meningkatkan probabilitas sebesar b₁" — itu interpretasi OLS! Dalam logistic regression, b₁ adalah perubahan log-odds, bukan perubahan probabilitas. Untuk probabilitas, lo harus hitung marginal effect di titik tertentu.

4
Evaluasi Model: Confusion Matrix & ROC/AUC

Logistic regression nggak punya R² tradisional (karena bukan OLS). Untuk mengevaluasi performa model, kita pakai beberapa tools:

Confusion Matrix:

Predicted = 1Predicted = 0
Actual = 1 True Positive (TP) False Negative (FN)
Actual = 0 False Positive (FP) True Negative (TN)
Accuracy = (TP + TN) / Total
Precision = TP / (TP + FP) // Dari yang diprediksi positif, berapa yang benar?
Recall (Sensitivity) = TP / (TP + FN) // Dari yang actual positif, berapa yang ketangkap?
Specificity = TN / (TN + FP)

ROC Curve & AUC: ROC curve memplot True Positive Rate (recall) vs False Positive Rate (1 − specificity) di berbagai threshold. AUC (Area Under Curve) merangkum keseluruhan performa: AUC = 0.5 → model sama kayak tebak random; AUC = 1.0 → perfect classifier.

Likelihood Ratio (LR) Test: Membandingkan log-likelihood restricted model vs unrestricted model. LR = −2 × (LLrestricted − LLunrestricted), distributed chi-squared. Kalau LR signifikan → unrestricted model lebih baik.

CFA Context: Logistic regression juga punya Pseudo-R² di software output, tapi hati-hati — pseudo-R² hanya bisa dipakai untuk membandingkan spesifikasi berbeda dari model yang sama (dataset sama), bukan untuk membandingkan model dari dataset berbeda.
5
Logistic Regression dalam Machine Learning

Logistic regression adalah building block utama untuk binary classification di machine learning dan neural networks. Contoh penerapan di finance:

• Credit scoring: Prediksi apakah nasabah akan default (Y = 1) atau tidak (Y = 0) berdasarkan rasio keuangan.

• Sentiment classification: Logistic regression + NLP untuk mengklasifikasi sentimen teks keuangan (press release, earnings call) menjadi positif (1) atau negatif (0).

• Fraud detection: Klasifikasi transaksi sebagai fraud vs legitimate.

Koneksi ke ML: Di CFA, logistic regression menjadi jembatan antara "traditional statistics" (LM1–LM4) dan "machine learning" (LM5+). Prinsip train/test split, overfitting, dan model evaluation yang lo pelajari di sini akan muncul terus di supervised learning.

Format CFA L2 · 4 soal · ~30 menit · Jawab dulu sebelum lihat penjelasan
Case Vignette
Rizal Prasetyo, CFA, adalah quantitative analyst di PT Nusantara Asset Management, Jakarta. Ia membangun model logistic regression untuk memprediksi probabilitas financial distress (Y = 1 jika distress, Y = 0 jika tidak) pada 200 emiten di IDX menggunakan data tahunan. Model estimasinya:

ln(P/(1−P)) = −1.20 + 2.50 × DE − 0.80 × ROE + 1.10 × LEV

dimana DE = debt-to-equity ratio, ROE = return on equity, LEV = financial leverage ratio. Semua koefisien signifikan pada level 5%.

Rizal juga menguji model menggunakan test sample 50 perusahaan dengan threshold 0.5. Hasilnya: TP = 12, FP = 5, FN = 3, TN = 30. Log-likelihood unrestricted model = −98.5, log-likelihood intercept-only model = −118.2.

Untuk perusahaan PT Maju Bersama (DE = 0.55, ROE = 0.14, LEV = 0.85), Rizal diminta menghitung probabilitas distress.
Question 1 of 4
Odds ratio untuk variabel DE (debt-to-equity) sebesar 2.50 diinterpretasikan sebagai:
B adalah jawaban yang benar. Dalam logistic regression, koefisien b₁ = 2.50 berarti kenaikan 1 unit DE meningkatkan log-odds sebesar 2.50, ceteris paribus. Odds ratio = e2.50 = 12.18, artinya odds distress menjadi 12.18× lebih tinggi. Opsi A salah — koefisien logistic regression bukan perubahan probabilitas langsung (itu interpretasi LPM). Opsi C salah — ini model distress, bukan return model.
Question 2 of 4
Probabilitas financial distress untuk PT Maju Bersama (DE = 0.55, ROE = 0.14, LEV = 0.85) adalah closest to:
A adalah jawaban yang benar. Step 1 — hitung logit (z): z = −1.20 + 2.50(0.55) − 0.80(0.14) + 1.10(0.85) = −1.20 + 1.375 − 0.112 + 0.935 = 0.998. Step 2 — masukkan ke sigmoid: P = 1/(1 + e−0.998) = 1/(1 + 0.369) = 1/1.369 ≈ 73.1%. Opsi B (50%) sesuai z = 0 — terlalu rendah. Opsi C (26.9%) sesuai z ≈ −1.0 — arah berlawanan. Kunci: selalu hitung z dulu, lalu masukkan ke sigmoid function.
Question 3 of 4
Berdasarkan confusion matrix test sample Rizal, precision model-nya adalah:
B adalah jawaban yang benar. Precision = TP/(TP + FP) = 12/(12 + 5) = 12/17 = 70.6%. Precision mengukur: dari semua yang model prediksi sebagai "distress" (positive), berapa persen yang benar-benar distress? Opsi A menghitung recall (sensitivity) = TP/(TP + FN) = 12/15 = 80%. Opsi C menghitung accuracy = (TP + TN)/Total = 42/50 = 84%.
Question 4 of 4
Rizal melakukan Likelihood Ratio test membandingkan model lengkap vs intercept-only model. LR test statistic dan kesimpulannya (critical χ² df=3, α=5% adalah 7.815) adalah:
C adalah jawaban yang benar. LR = −2 × (LLrestricted − LLunrestricted) = −2 × (−118.2 − (−98.5)) = −2 × (−19.7) = 39.4. Karena 39.4 > 7.815 (critical value), kita tolak H₀ bahwa restricted (intercept-only) model lebih baik. Kesimpulan: setidaknya satu koefisien ≠ 0, variabel independen secara bersama signifikan. Opsi A salah karena 19.70 adalah selisih log-likelihood, bukan LR statistic (lupa dikali −2). Opsi B salah angkanya.
