1
Method Selection: Supervised vs Unsupervised, Data Type, Data Size

Model training — seperti dijelaskan di Exhibit 20 kurikulum CFA — adalah proses iteratif yang melibatkan tiga fase berurutan: Method Selection → Performance Evaluation → Tuning. Sebelum masuk ke detail tiap fase, perlu dipahami bahwa fundamentalnya ML training adalah fitting a system of rules on a training dataset to reveal patterns. "Fitting" di sini menggambarkan seberapa baik model bisa di-generalize ke data baru (out-of-sample).

Metodologi ML training untuk structured dan unstructured data pada dasarnya sama — karena unstructured data (teks, gambar, audio) sudah di-process di tahap data preparation menjadi structured matrix (ingat: DTM untuk teks di Day 13). Setelah data siap, pemilihan metode menjadi langkah kritis pertama.

1.1 Faktor Pertama: Supervised vs Unsupervised

Supervised ML — data training memiliki ground truth (target variable Y yang diketahui). Model belajar memetakan X → Y. Contoh: prediksi default obligasi high-yield dari data issuer. Metode tipikal: regression, ensemble trees, SVMs, neural networks.

Unsupervised ML — tidak ada target variable. Model menemukan struktur sendiri dari data. Lebih menantang karena tidak ada "jawaban benar" untuk di-cocokkan. Contoh: clustering lembaga keuangan berdasarkan atribut finansial. Metode tipikal: dimensionality reduction (PCA), clustering (k-means, hierarchical), anomaly detection.

1.2 Faktor Kedua: Type of Data

1.3 Faktor Ketiga: Size of Data

Ada dua dimensi ukuran dataset yang berinteraksi menentukan metode optimal:

  • Number of instances (observations) — jumlah baris
  • Number of features — jumlah kolom (variabel independen)

SVM bekerja baik pada wide datasets (10.000–100.000 features, instances lebih sedikit) — misal text classification dengan BOW yang memiliki puluhan ribu token. NN bekerja baik pada long datasets (instances ≫ features) — misal image recognition pada jutaan foto dengan ratusan ribu pixel. Di praktik, dataset sering campuran numeric + text, sehingga hasil beberapa metode dikombinasikan (stacking): misalnya, prediksi sentiment dari logistic regression dipakai sebagai feature tambahan di CART yang juga pakai data finansial structured.

Knowledge Check 1.1
Anda punya dataset 200 perusahaan IDX (instances) dengan 35.000 features (tokens dari pidato CEO di laporan tahunan + rasio finansial). Target: prediksi return 12-bulan ke depan (binary: beat market Y/N). Metode mana yang paling cocok?
SVM adalah pilihan paling cocok. Dataset ini "wide" — 35.000 features jauh melebihi 200 instances. SVM bekerja sangat baik pada wide datasets karena margin maximization mampu menangani dimensi tinggi tanpa overfitting berlebihan (asal di-regularize). Neural networks akan overfit dengan data se-"longaran" ini; CART juga rentan overfit. Alternatif: penalized regression (LASSO) untuk automatic feature selection.
2
Dataset Partitioning: Train/Validation/Test (60:20:20) & Class Imbalance

Sebelum training dimulai, master dataset untuk supervised learning wajib dipecah menjadi tiga subset yang non-overlapping. Ini adalah salah satu aturan paling fundamental di ML — dan salah satu sumber kesalahan paling umum di praktik.

2.1 Tiga Subset dan Fungsinya
Recommended Split — CFA Curriculum Master dataset = 100%

├── Training sample ≈ 60% → fit model PARAMETERS (coefficients, weights)
├── Validation / CV sample ≈ 20% → tune HYPERPARAMETERS; pilih model terbaik
└── Test sample ≈ 20% → final OUT-OF-SAMPLE evaluation (touched once) Split ratio 60:20:20 adalah default CFA; bisa juga 70:15:15 untuk dataset kecil. Unsupervised learning tidak butuh split karena tidak ada target variable.

Training dan validation dianggap "in-sample" karena keduanya dipakai dalam proses modeling. Test adalah "out-of-sample" — bertindak sebagai proxy data baru yang belum pernah dilihat model.

Golden Rule — Jangan Bocor Test Set

Test set hanya boleh "di-touch" sekali — pada evaluasi final. Kalau lo tune hyperparameter berdasarkan performa test set, test set secara efektif menjadi validation set kedua, dan reported test error under-estimates true out-of-sample error. Ini kesalahan yang sering ditanyakan di exam CFA L2.

2.2 Teknik Random Sampling

Pembagian biasanya dilakukan via random sampling (mis. k-fold method) untuk memastikan ketiga subset representatif dari distribusi populasi. Di beberapa kasus (mis. data dengan struktur kelas tidak seimbang), stratified sampling digunakan: setiap subset mempertahankan proporsi kelas yang sama dengan master dataset.

2.3 Class Imbalance Problem

Class imbalance terjadi ketika jumlah instance di satu kelas jauh lebih besar dari kelas lain. Contoh klasik di finance: default prediction — issuer obligasi BB+/Ba1 hingga B+/B1 memiliki tingkat default historis hanya ~3–5%. Jadi, dari 1.000 bond, mungkin hanya 40 yang default (Class 1), sisanya 960 tidak (Class 0).

Model ML yang naif dengan mengoptimalkan accuracy akan memprediksi "tidak default" untuk semua observasi → accuracy 96%, tapi recall = 0%. Model ini useless: 100% kasus default lolos (Type II error sempurna).

Exhibit 2.1 — Teknik Balancing untuk Imbalanced Data
TeknikBagaimanaKapan Dipakai
Undersampling majorityBuang random sebagian Class 0 sampai rasio seimbangDataset besar; buang data masih acceptable
Oversampling minorityReplicate random Class 1 (dengan atau tanpa replacement)Dataset kecil; tidak mau buang data
Synthetic generation (SMOTE)Generate observasi baru minoritas dengan interpolasi antar-observasiAdvanced; saat oversampling biasa overfit
Implikasi ke Metric Pilihan

Ketika data imbalanced, accuracy menyesatkan. Metric yang lebih tepat: F1 score (harmonic mean precision & recall), AUC (threshold-independent), atau Matthews Correlation Coefficient (MCC). F1 khususnya penting karena ia memberikan bobot sama pada precision dan recall — dan akan langsung hancur kalau salah satunya buruk.

Knowledge Check 2.1
Tim fraud detection mempunyai 1.000.000 transaksi kartu kredit dengan hanya 2.000 (0.2%) yang fraudulent. Model mereka mencapai accuracy 99.8%. Apakah model ini baik? Bagaimana cara mengevaluasinya lebih baik?
Accuracy 99.8% tidak informatif. Model yang memprediksi "non-fraud" untuk semua transaksi otomatis dapat accuracy 99.8% (= 1 − 0.002). Evaluasi yang lebih baik: (1) Hitung precision dan recall — khususnya recall karena cost of FN (fraud yang lolos) sangat tinggi. (2) Hitung F1. (3) Plot ROC curve dan AUC. (4) Pertimbangkan undersampling Class 0 atau SMOTE Class 1 sebelum training supaya model belajar pola fraud, bukan hanya prediksi mayoritas.
3
Bias–Variance Tradeoff & Model Fitting Scenarios

Dalam CFA L2 curriculum, model fitting diilustrasikan melalui Exhibit 21 — tiga skenario pada classification task dengan dua kelas (circles dan triangles):

  • Underfit (kiri): decision boundary terlalu sederhana (mis. garis lurus), 4 misclassifications pada training data. Model tidak menangkap pattern.
  • Overfit (tengah): decision boundary "S-shaped" sempurna di training (0 error), tetapi tidak akan generalize — bentuknya menghafal noise.
  • Good fit (kanan): decision boundary smooth dengan 1 misclassification; generalize dengan baik ke data baru.
3.1 Dekomposisi Error
Total Error Decomposition Total Error = Bias² + Variance + Irreducible Noise

Bias — kesalahan dari asumsi model yang terlalu simplistik
Variance — kesalahan dari model terlalu sensitif terhadap fluktuasi training data
Noise — error tak tereduksi (inherent randomness di data) Bias–variance tradeoff: kita tidak bisa menghilangkan keduanya sekaligus, tapi bisa minimalisasi total error.
3.2 Gejala Underfit vs Overfit (Exam-Critical)
Exhibit 3.1 — Mendiagnosis Bias vs Variance
Training ErrorCV ErrorGap
Underfitting (high bias)TinggiTinggiKeduanya tinggi; gap kecil
Overfitting (high variance)RendahTinggiGap besar: CV ≫ train

3.3 Penyebab Utama Model Fitting Errors

Dataset size: dataset kecil sering menyebabkan underfit karena tidak cukup untuk mengekspos pattern. Menambah data training membantu mengurangi variance error, tapi tidak bisa mengatasi bias error (model tetap terlalu simple).

Number of features: terlalu sedikit features → underfit (informasi kurang). Terlalu banyak features → overfit (low degrees of freedom, model "menghafal" noise). Feature selection (chi-square, mutual information) adalah kunci mitigasi overfitting pada high-dimensional data.

Feature engineering cenderung mengatasi underfitting — membuat features yang lebih informatif (mis. ratios, interaction terms, log transforms) mengelevasi pattern yang tersembunyi di raw data.

Occam's Razor dalam ML

Prinsip Occam's razor — solusi paling sederhana biasanya paling benar — diterapkan di ML sebagai overfitting penalty. Model dengan kompleksitas berlebih (koefisien banyak, tree sangat dalam, NN banyak layer) di-penalize agar lebih sederhana. Teknik: L1/L2 regularization, tree pruning, early stopping, dropout di NN.

4
Cross-Validation: k-Fold, LOOCV, dan Walk-Forward untuk Time Series

Seperti dijelaskan di kurikulum, ada dua strategi utama untuk mitigasi overfitting:

  1. Complexity reduction (regularization) — penalti pada kompleksitas model (L1/L2, pruning, dropout).
  2. Proper data sampling via cross-validation — estimate out-of-sample error langsung dari data.
4.1 k-Fold Cross-Validation

Teknik CV paling umum. Algoritma:

  1. Shuffle data (kecuali test sample dan fresh data yang tidak boleh digunakan).
  2. Bagi data menjadi k equal sub-samples (folds). Biasanya k = 5 atau 10.
  3. Untuk tiap iterasi i = 1 ... k: gunakan (k-1) folds sebagai training, 1 fold sebagai validation. Catat validation error E_i.
  4. E_CV (mean E_val) = rata-rata k validation errors ≈ estimate of E_out (out-of-sample error).
k-Fold Cross-Validation Error E_CV = (1/k) · Σ E_val,i for i = 1, 2, ..., k Tiap observasi dipakai (k-1) kali di training set, 1 kali di validation set. Mengurangi sampling bias tanpa mengorbankan ukuran training set.

Keuntungan: (1) pemanfaatan data maksimal, tiap observasi berkontribusi ke baik training maupun validation, (2) estimasi E_out lebih stabil dari single-split, (3) cocok untuk dataset kecil/sedang. Kerugian: (1) k kali lebih mahal computationally, (2) tidak cocok untuk time series.

4.2 Varian Lain: LOOCV dan Stratified

Leave-One-Out CV (LOOCV): k = n (jumlah observasi). Tiap iterasi, 1 observasi untuk validation, n-1 untuk training. Sangat mahal tapi minim bias. Jarang dipakai untuk big data.

Stratified k-Fold: setiap fold mempertahankan proporsi kelas yang sama dengan dataset asli. Penting untuk imbalanced data — mencegah fold yang kebetulan tidak memuat kelas minoritas.

4.3 Mengapa k-Fold Gagal untuk Time Series

Kurikulum CFA secara eksplisit mengingatkan: "k-fold cross-validation cannot be used with time-series data, where only the most recent data can reasonably be used for model validation."

Alasan: random shuffling memecah urutan waktu. Model bisa dapat akses ke data "masa depan" saat training, lalu di-validate dengan data "masa lalu" — ini adalah look-ahead bias. Dalam konteks trading/forecasting, model akan tampak lebih akurat dari kenyataannya.

Exhibit 4.1 — Walk-Forward / Expanding Window Validation
IterasiTraining WindowValidation Window

Training window bisa expanding (grows) atau rolling (fixed length, sliding). Validation selalu setelah training window — menghormati panah waktu. Teknik ini juga dipakai luas di quantitative strategy backtesting.

Knowledge Check 4.1
Tim Anda mengklasifikasi 10.000 earnings call transcripts (2015–2025) menjadi bullish/bearish sentiment. Anda berencana k-fold dengan k=10 random shuffle. Apakah ini tepat? Jelaskan.
Tepat — tapi hati-hati. Kalau tujuan hanya mengklasifikasi sentiment (cross-sectional task: "apakah kalimat ini positif?"), maka k-fold dengan random shuffle OK — karena sentiment adalah properti intrinsik teks, bukan time-dependent. Tetapi kalau tujuan adalah menggunakan sentiment untuk memprediksi return saham ke depan (time-series task), maka k-fold random shuffle akan menciptakan look-ahead bias. Pakailah walk-forward validation: train model di 2015–2020, test di 2021, roll ke 2021 training, test 2022, dst.

Performance Evaluation: Confusion Matrix, Precision, Recall, F1, ROC/AUC, RMSE

Setelah model di-train, langkah kritis berikutnya adalah mengukur performa (goodness of fit). Metrik yang dipakai berbeda untuk classification vs regression.

5.1 Confusion Matrix & Error Analysis

Untuk binary classification, confusion matrix merupakan grid 2×2 yang merangkum empat metrik dasar. Misalkan Class "1" = "defective" dan Class "0" = "not defective":

Exhibit 5.1 — Confusion Matrix
Actual = 1Actual = 0
Predicted = 1TP (True Positive)FP (False Positive) — Type I Error
Predicted = 0FN (False Negative) — Type II ErrorTN (True Negative)
Performance Metrics — Formulae (CFA L2 Wajib Hafal) Precision (P) = TP / (TP + FP)
Recall (R) = TP / (TP + FN) = True Positive Rate (TPR)
Accuracy = (TP + TN) / (TP + FP + TN + FN)
F1 Score = 2 · P · R / (P + R)
False Positive Rate (FPR) = FP / (TN + FP) Precision = dari yang diprediksi positif, % yang benar. Recall = dari yang aktual positif, % yang berhasil ditangkap model.

5.3 Kapan Precision vs Recall Lebih Penting?
Trade-off Precision vs Recall

Precision penting ketika cost of FP tinggi. Contoh: produk mahal (mis. microchip) yang predicted defective → di-scrap, padahal sebenarnya OK. FP bikin rugi finansial. Atau: sistem spam filter — FP artinya email penting masuk spam folder.

Recall penting ketika cost of FN tinggi. Contoh: medical test kanker — FN artinya pasien sakit tapi didiagnosis sehat (bisa fatal). Atau: fraud detection — FN artinya transaksi fraudulent lolos. Di credit default prediction bank, FN (approve kredit ke nasabah yang akan default) = kerugian besar → bank prioritaskan recall.

5.4 ROC Curve & AUC

Receiver Operating Characteristic (ROC) curve = plot True Positive Rate (y-axis) vs False Positive Rate (x-axis) untuk berbagai classification thresholds (mis. p-threshold di logistic regression: 0.3, 0.4, 0.5, 0.6, ...).

Untuk setiap threshold, hitung TP, FP, TN, FN, lalu plot titik (FPR, TPR). Menghubungkan titik-titik menghasilkan kurva ROC. Area Under the Curve (AUC) adalah metrik tunggal yang merangkum ROC:

Exhibit 5.3 — Interpretasi AUC
AUCInterpretasi
1.0Perfect classifier
0.9–0.99Excellent — kurva sangat konveks
0.7–0.9Good
0.5–0.7Poor — mendekati random
0.5Random guessing — garis diagonal
< 0.5Lebih buruk dari random (flip prediksi supaya > 0.5)

Keunggulan AUC: threshold-independent. Model dengan AUC tinggi akan memiliki decision ranking yang baik — memberi probabilitas tinggi untuk true positives dan rendah untuk true negatives, tanpa bergantung pada cutoff.

5.5 RMSE untuk Regression

Untuk prediksi continuous (regression), metrik standarnya adalah Root Mean Squared Error:

Root Mean Squared Error RMSE = √[ Σ(Predicted_i − Actual_i)² / n ] for i = 1, 2, ..., n RMSE kecil = model lebih akurat. Karena squaring, RMSE sensitif terhadap outlier. Alternatif: MAE (Mean Absolute Error), MAPE (Mean Absolute Percentage Error).
Knowledge Check 5.1
Model prediksi default KPR menghasilkan confusion matrix: TP=25, FP=40, FN=10, TN=925 (total 1000 observasi). Hitung precision, recall, F1, dan accuracy. Apa komentar Anda?
Precision = 25/65 = 0.385; Recall = 25/35 = 0.714; F1 = 2·0.385·0.714/(0.385+0.714) = 0.500; Accuracy = 950/1000 = 0.95. Komentar: accuracy 95% terdengar bagus tapi menyesatkan — dataset sangat imbalanced (3.5% default). F1 = 0.50 lebih jujur menunjukkan model biasa saja. Model punya recall relatif baik (menangkap 71% defaulters) tapi precision rendah (dari yang diprediksi akan default, hanya 38.5% benar). Untuk bank, trade-off ini mungkin masih acceptable karena mereka lebih takut FN daripada FP — lebih baik over-reject kredit (FP) daripada approve yang akan default (FN).

6.2 Grid Search

Karena tidak ada rumus umum untuk mengestimasi hyperparameter yang optimal, dipakai teknik heuristik. Grid search adalah metode paling populer: spesifikasikan sebuah "grid" nilai calon untuk setiap hyperparameter, lalu train model pada tiap kombinasi menggunakan cross-validation, dan pilih kombinasi yang meminimumkan CV error dengan syarat training error dan CV error relatif sama (bukan overfit).

Grid Search — Contoh untuk Random Forest Grid:
n_trees ∈ {100, 200, 500}
max_depth ∈ {5, 10, 20}
min_samples ∈ {5, 10, 20}

Total: 3 × 3 × 3 = 27 kombinasi
Untuk tiap kombinasi: run 10-fold CV → catat E_CV
Pilih kombinasi dengan E_CV terkecil dan |E_train − E_CV| kecil Alternatif modern: randomized search (sample random subset grid) dan Bayesian optimization (lebih efisien untuk high-dim hyperparameter space).
6.3 Fitting Curve (Exhibit 26 Kurikulum)

Fitting curve adalah plot error (y-axis) vs hyperparameter value (x-axis). Untuk regularization λ di LASSO/Ridge:

Exhibit 6.2 — Tiga Zona pada Fitting Curve
ZoneλE_trainE_CVDiagnosis

Di zone kiri, model memiliki semua fitur aktif tanpa penalty — menghafal data training (low bias, high variance). Di zone kanan, regularization terlalu keras memaksa sebagian besar koefisien ke nol — model terlalu simplistik (high bias, low variance). Di tengah, trade-off optimal: cukup regularization untuk eliminasi features yang tidak relevan, tetapi tidak begitu keras sehingga kehilangan informasi.

6.4 Tuning Decisions Based on Diagnosis
Tindakan untuk Each Diagnosis

High bias (underfit): tambahkan lebih banyak features, tambah feature engineering, reduce regularization, tingkatkan complexity model (mis. NN lebih dalam, tree lebih dalam). Menambah data training tidak akan membantu underfit.

High variance (overfit): kumpulkan lebih banyak training data, reduce number of features, tingkatkan regularization, simplify model. Data augmentation juga bisa membantu.

6.5 Ceiling Analysis

Ceiling analysis adalah teknik untuk complex models yang terdiri dari beberapa sub-models/komponen dalam pipeline. Tujuan: mengidentifikasi sub-model mana yang paling membatasi overall performance sehingga prioritas tuning dialokasikan di tempat yang tepat.

Contoh dari kurikulum: stock market prediction model yang membutuhkan (a) historical stock data, dan (b) news articles. Sub-model di pipeline: sentiment classifier → feeds into return forecasting model.

Ceiling Analysis — Prosedur Baseline: End-to-end pipeline accuracy = 72%

Step 1: Ganti output sentiment classifier dengan GROUND TRUTH
— Run pipeline → new accuracy = ?
Kasus A: naik ke 88% (+16 pp) → sentiment classifier adalah bottleneck, tune dulu
Kasus B: naik ke 73% (+1 pp) → sentiment bukan bottleneck; tune forecasting model

Step 2: Ganti output forecasting model dengan GROUND TRUTH
— Lihat lift untuk konfirmasi Prinsip: alokasikan effort tuning ke komponen yang menunjukkan lift terbesar saat disubstitusi oracle.
7
Model Interpretability & Backtesting ML Signals
7.1 Interpretability: Black Box vs White Box

Model ML berbeda-beda dalam tingkat interpretability:

  • White box (highly interpretable): linear/logistic regression, decision trees (single). Setiap koefisien/split node dapat dijelaskan.
  • Gray box: random forest, gradient boosting. Feature importance scores bisa dihitung, tetapi cara interaksi features antar-tree sulit dilihat.
  • Black box (low interpretability): deep neural networks, SVMs dengan kernel non-linear. Akurasi tinggi tapi "mengapa" model memutuskan X tidak jelas.

Dalam praktik finance, interpretability penting karena:

  1. Regulator (OJK, SEC) menuntut transparansi model (mis. model risk management).
  2. Klien institusi menuntut penjelasan keputusan investasi.
  3. Portfolio manager perlu understand model untuk percaya ke signalnya.

Teknik untuk improve interpretability di black-box models: SHAP values, LIME, partial dependence plots, feature importance rankings. Tidak wajib hafal detail teknisnya untuk CFA L2, tapi perlu paham konsepnya.

7.2 Backtesting ML Trading Signals

Ketika ML model dipakai untuk generate trading signals, backtesting menjadi critical step sebelum live deployment. Best practices:

Lima Prinsip Backtesting ML Signals

1. Walk-forward validation — jangan k-fold random shuffle di time series. Train di data historis, test di periode setelahnya.

2. Out-of-sample period cukup panjang — minimum 1 cycle market yang complete (mis. bull-bear-bull) untuk test robustness di regime yang berbeda.

3. Transaction costs, slippage, market impact — masukkan ke simulasi PnL. Signal yang tampak profitabel di gross return sering jadi loss-making net of costs.

4. Hati-hati dengan data snooping bias — kalau lo coba 100 strategy variations, beberapa akan tampak significant hanya karena kebetulan. Pakai Bonferroni correction atau out-of-sample test yang benar-benar baru.

5. Survivorship bias — pastikan data training tidak hanya berisi saham yang "survive" hingga hari ini (universe harus include delistings, bankruptcies).

Knowledge Check 7.1
ML strategy Anda menunjukkan Sharpe ratio 2.5 di backtest tahun 2018–2024. Live deployment di 2025 menghasilkan Sharpe 0.3. Apa kemungkinan penyebab divergence?
Beberapa kemungkinan: (1) Overfitting — model memorized pattern periode 2018–2024 yang tidak berulang. (2) Look-ahead bias di backtest (mis. pakai data yang tidak tersedia real-time). (3) Transaction costs/slippage yang tidak dimodelkan dengan akurat — strategi dengan turnover tinggi sering kolaps post-cost. (4) Regime change — karakteristik pasar 2025 berbeda fundamental (mis. rezim suku bunga, volatility). (5) Capacity/market impact — saat scale up dana, eksekusi menggerakkan harga. (6) Survivorship bias di training universe. Solusi: re-train pada data yang mencakup 2025, tambah walk-forward validation, masukkan realistic trading costs, dan monitor rolling Sharpe untuk deteksi dini.

Vignette — Quantum Asset Management
Arif Rahman, CFA, adalah Head of Quantitative Research di Quantum Asset Management (QAM), sebuah hedge fund berbasis Jakarta. Tim Arif sedang mengembangkan model ML untuk memprediksi arah pergerakan 1-bulan ke depan dari saham-saham LQ45 (binary classification: naik/turun).

Dataset master berisi 8.500 observasi bulanan sepanjang 2015-01 hingga 2025-03 (45 saham × ~190 bulan), dengan 48% label "naik" dan 52% "turun" (relatif balanced). Features mencakup 15 rasio finansial, 8 technical indicators, sentiment score dari headline berita (sub-model terpisah dengan accuracy 68%), dan 3 macro variables. Total 27 features per observasi.

Arif memilih gradient boosting classifier dengan hyperparameter utama: n_trees, max_depth, learning_rate, dan min_samples_leaf. Dia membagi data sebagai berikut: 5.100 observasi training (60%), 1.700 validation (20%), 1.700 test (20%) — dengan random shuffle.

Hasil initial training menunjukkan: training accuracy = 0.88, CV accuracy = 0.60, test accuracy = 0.59. Confusion matrix pada test set: TP = 380 (actual naik, predicted naik), FP = 320, FN = 440, TN = 560.

Analis junior Siti Nurhaliza memproposalkan dua perbaikan: (1) gunakan k-fold CV dengan k=10 pada training+validation set combined untuk estimasi lebih stabil, dan (2) lakukan ceiling analysis dengan mengganti output sentiment classifier dengan ground truth untuk melihat apakah sentiment adalah bottleneck. Hasil ceiling analysis: saat sentiment diganti oracle, test accuracy naik dari 0.59 ke 0.62.
Soal 1 / 6
Kritik paling signifikan terhadap metodologi split dataset Arif adalah:
B adalah jawaban yang benar. Ini adalah forecasting problem yang time-dependent (memprediksi 1-month forward return). Random shuffle merusak urutan waktu — data 2024 bisa masuk training, data 2016 ke test → look-ahead bias. Solusi: walk-forward validation (train up to month T, test month T+1, roll forward). A salah karena 60:20:20 adalah rekomendasi CFA standard dan bukan faktor utama di sini. C salah karena 8.500 observasi cukup untuk gradient boosting dengan 27 features; tidak ada rule "50.000 minimum".
Soal 2 / 6
Berdasarkan gap training accuracy (0.88) vs CV accuracy (0.60), model menunjukkan:
B adalah jawaban yang benar. Classical overfitting symptom: training error kecil (accuracy 88%) sementara CV error jauh lebih besar (accuracy 60% → error 40%). Gap 28 percentage points terlalu lebar. Tindakan mitigasi overfit: (1) data augmentation / kumpulkan lebih banyak training data, (2) feature selection untuk reduce number of features, (3) tingkatkan regularization — untuk gradient boosting berarti kurangi max_depth, naikkan min_samples_leaf, atau turunkan n_trees/learning_rate. A salah (underfit justru training error tinggi juga). C salah (gap 28% menunjukkan masalah, bukan good fit).
Soal 3 / 6
Hitung precision, recall, F1, dan accuracy dari confusion matrix test set. Pilihan mana yang benar?
C adalah jawaban yang benar. Dari matrix: TP=380, FP=320, FN=440, TN=560, Total=1700.
Precision = TP/(TP+FP) = 380/700 = 0.5429
Recall = TP/(TP+FN) = 380/820 = 0.4634
F1 = 2·P·R/(P+R) = 2·0.5429·0.4634/(0.5429+0.4634) = 0.5031/1.006 = 0.500
Accuracy = (TP+TN)/Total = (380+560)/1700 = 940/1700 = 0.553
A salah (Precision hitung keliru). B salah (keduanya keliru). Observasi tambahan: accuracy 55% hanya sedikit di atas coin flip (52% "turun" baseline), dan F1 0.50 menunjukkan model marginal.
Soal 4 / 6
QAM ingin memakai signal ini untuk strategi long-short. Manajer portfolio lebih khawatir "membeli saham yang justru turun" (false buy signal) daripada "melewatkan saham yang naik". Metric mana yang paling penting untuk dimonitor?
A adalah jawaban yang benar. "Membeli saham yang turun" = false positive (predicted naik = buy signal, actual turun). Cost of FP tinggi → precision penting. Precision = TP/(TP+FP) mengukur, dari semua saham yang kita beli berdasarkan signal, berapa % yang benar-benar naik. Dengan precision 0.543, hampir 46% signal buy QAM akan rugi — mungkin tidak acceptable. B salah karena recall peduli tentang missed opportunities (saham naik yang tidak dibeli), yang bukan concern manajer di sini. C salah karena accuracy imbalanced-aware saat class distribution tidak seimbang, dan di sini bukan metric terbaik untuk asymmetric cost.
Soal 5 / 6
Komentari proposal Siti tentang k-fold CV dengan k=10:
C adalah jawaban yang benar. Sama seperti Soal 1, data QAM adalah time-series forecasting (memprediksi return 1-bulan ke depan). Random k-fold shuffle membuat fold bisa berisi campuran periode 2016, 2020, dan 2024 — jadi model melihat "masa depan" saat training. Yang benar untuk data time-dependent: (i) time-series k-fold (tiap fold adalah block periode berurutan, tanpa shuffle), atau (ii) walk-forward validation. A salah karena mengabaikan time structure. B salah karena k-fold bisa dipakai untuk classification (bukan hanya regression); yang membuatnya tidak cocok di sini adalah time-series nature, bukan classification vs regression.
Soal 6 / 6
Berdasarkan ceiling analysis (accuracy naik dari 0.59 ke 0.62 saat sentiment classifier diganti oracle), kesimpulan paling tepat tentang prioritas tuning adalah:
B adalah jawaban yang benar. Interpretasi ceiling analysis: kalau mengganti sub-komponen dengan ground truth hanya memberi lift kecil (3 pp), maka sub-komponen itu BUKAN bottleneck — bahkan dengan sentiment sempurna, overall accuracy masih di 62%. Artinya 38% error lainnya berasal dari bagian lain pipeline: mungkin main model overfit (gap training 0.88 vs CV 0.60 sudah memberi petunjuk), feature set tidak cukup discriminative, atau target problem secara intrinsik sulit (pasar memang sulit diprediksi). Prioritas tuning: main gradient boosting classifier (reduce overfitting), feature engineering baru, atau consider model architecture lain. A salah karena menafsirkan ceiling analysis terbalik. C salah — ceiling analysis mengidentifikasi bottleneck, bukan memberitahu lo untuk berhenti.
