Cointegration & ARCH/GARCH

Time-Series Analysis — Subtopic 2 of 2

⏱ 60 min total
📖 30 min theory
📝 30 min practice
⚖️ Exam weight ~6%
Learning Objectives — Hari ini lo harus bisa:
  • Bedakan lima skenario unit-root saat meregresikan dua time series dan jelaskan kapan hasil regresi valid
  • Jelaskan konsep cointegration dan langkah-langkah Engle–Granger test untuk cointegration
  • Terapkan ARCH(1) test pada residual time-series model dan interpretasikan implikasinya untuk standard error
  • Gunakan koefisien ARCH untuk memprediksi variance error periode berikutnya dan bedakan ARCH(p) dari GARCH
Theory · 30 min
1
Regressing Two Time Series: 5 Skenario Unit Root

Sebelum meregresikan satu time series pada time series lain, lo wajib test unit root di kedua series pakai Dickey–Fuller. Kalau salah satu punya unit root, error term regresi nggak akan covariance stationary dan hasilnya bisa spurious (kelihatan signifikan padahal nggak bermakna).

SkenarioY (dep)X (indep)Regresi valid?
1stationarystationary✓ Ya, OLS valid
2unit rootstationary✗ Tidak
3stationaryunit root✗ Tidak
4unit rootunit root, tidak cointegrated✗ Spurious
5unit rootunit root, cointegrated✓ Long-run only
Key Insight: Untuk skenario 2 & 3 (mismatch), bahkan first-differencing nggak selalu menyelamatkan — karena hubungan antara "level" dan "change" itu fundamental. Untuk skenario 4, hasil regresi sepenuhnya spurious. Hanya skenario 1 dan 5 yang bisa diandalkan.
2
Cointegration — Hubungan Jangka Panjang

Dua time series dikatakan cointegrated kalau keduanya sama-sama punya unit root, tapi ada hubungan ekonomi/finansial jangka panjang yang membuat mereka nggak terpisah tanpa batas. Mereka bergerak bersama dalam jangka panjang — share a common trend.

Contoh klasik: Harga saham emiten tambang dan harga komoditas yang ditambangnya, atau nominal GDP dan penjualan perusahaan retail, atau suku bunga jangka pendek dan jangka panjang.

Key Insight: Cointegration ≠ correlation. Dua series bisa sangat berkorelasi dalam sampel tapi diverge tanpa batas seiring waktu (spurious). Cointegration mengharuskan adanya equilibrium relationship yang mean-reverting di residual.
⚠ Exam Trap

Bahkan kalau cointegrated (skenario 5), hasil regresi hanya valid untuk hubungan jangka panjang. Untuk dinamika jangka pendek dibutuhkan Error Correction Model (ECM) — ini di luar scope CFA tapi sering muncul di distraktor.

3
Engle–Granger Test — Langkah demi Langkah

Kalau kedua series punya unit root, kita harus test cointegration sebelum trust regresi. Engle–Granger prosedur:

Step 1: Estimasi regresi yt = b₀ + b₁xt + εt
Step 2: Ambil residual ε̂t, test unit root pada residual
Step 3: Gunakan Engle–Granger critical values, BUKAN DF standar
// H₀: residual punya unit root (tidak cointegrated)
// Hₐ: residual stationary (cointegrated)

Interpretasi hasil:

• Tolak H₀ → residual stationary → series cointegrated → regresi valid untuk long-run relationship.

• Gagal tolak H₀ → residual non-stationary → NOT cointegrated → regresi spurious, jangan dipakai.

⚠ Exam Trap

EG critical values lebih negatif dari DF biasa karena harus memperhitungkan ketidakpastian parameter b₀ dan b₁ yang diestimasi di Step 1. Pakai DF biasa untuk residual = over-rejection H₀ = false conclusion of cointegration.

4
ARCH(1) — Variance yang Bergantung Waktu

Sampai sekarang kita asumsikan homoskedastic — variance error konstan. Tapi di data keuangan sering terjadi volatility clustering: periode tenang diikuti periode tenang, periode volatil diikuti periode volatil. Ini autoregressive conditional heteroskedasticity (ARCH), ditemukan oleh Robert Engle (Nobel 2003).

ARCH(1) model: εt ~ N(0, a₀ + a₁ε²t−1)
// Variance error hari ini tergantung squared error kemarin
Test regression: ε̂²t = a₀ + a₁ε̂²t−1 + ut
// H₀: a₁ = 0 (no ARCH). Hₐ: a₁ ≠ 0 (ARCH exists)

Kalau a₁ signifikan:

(1) Standard error dari AR model tidak valid — pakai generalized least squares (GLS) untuk koreksi. (2) Variance error periode mendatang bisa diprediksi.

Forecast variance: σ̂²t+1 = â₀ + â₁ε̂²t
// Contoh: jika â₀=6.36, â₁=0.275, error kemarin=1%
// maka σ̂²t+1 = 6.36 + 0.275(1²) = 6.64
Key Insight: ARCH sangat umum di return saham, nilai tukar, dan harga komoditas. Kalau lo mengabaikannya, t-stat koefisien AR bisa overstated — variabel yang seharusnya nggak signifikan jadi kelihatan signifikan → keputusan investasi salah.
5
ARCH(p) dan GARCH — Generalisasi

ARCH(p): Variance error tergantung pada squared error dari p periode sebelumnya:

σ²t = a₀ + a₁ε²t−1 + a₂ε²t−2 + ... + apε²t−p

GARCH (Generalized ARCH): Seperti ARMA untuk variance — menggabungkan lagged squared errors dan lagged variance.

Limitasi GARCH: Parameter bisa sangat tidak stabil — hasil berubah drastis tergantung sampel dan initial guess parameter. Kandidat CFA harus hati-hati: GARCH itu powerful tapi "finicky". Selalu cek robustness terhadap perubahan sampel.

Konteks Praktis: GARCH(1,1) adalah standard industry untuk volatility forecasting di VaR modeling, options pricing, dan risk management. Formulanya: σ²t = a₀ + a₁ε²t−1 + b₁σ²t−1.

Practice · 30 min

Mini Vignette — PT Bumi Resources & Harga Batubara
4 soal · Cointegration & ARCH/GARCH
Vignette
Arief Wijaya, analis ekuitas di Mandiri Sekuritas, sedang meneliti hubungan antara harga saham PT Bumi Resources Tbk (BUMI) dan harga batubara Newcastle (dalam USD/ton). Menggunakan 120 observasi bulanan (Jan 2016 – Des 2025), Arief meregresikan ln(harga BUMI) pada ln(harga batubara):

ln(BUMIt) = −3.84 + 1.12 × ln(Coalt) + εt, R² = 0.81, t-stat β₁ = 22.6

Sebelum menyimpulkan, Arief menjalankan Dickey–Fuller test pada kedua series:
• ln(BUMI): DF t-stat = −1.68 (5% CV = −2.89)
• ln(Coal): DF t-stat = −1.45 (5% CV = −2.89)
• Residual regresi: DF t-stat = −3.92 (EG 5% CV = −3.34)

Arief juga mengestimasi AR(1) untuk return bulanan BUMI dan menjalankan ARCH(1) test pada residual kuadratnya:
ε̂²t = 0.0082 + 0.342 × ε̂²t−1 + ut, t-stat pada koefisien lag = 4.18, n = 119.
Bulan lalu, error term AR(1) = +8.0% (residual positif besar setelah earnings surprise).
Soal 1 / 4
Berdasarkan hasil Dickey–Fuller test pada kedua series, Arief paling tepat menyimpulkan bahwa:
B adalah jawaban yang benar. DF test pada ln(BUMI): t-stat = −1.68 > −2.89 (critical value). Gagal tolak H₀ → unit root ada. Sama untuk ln(Coal): −1.45 > −2.89 → unit root ada. Karena kedua series punya unit root, kita masuk ke skenario 4 atau 5 — harus test cointegration sebelum menyimpulkan apapun dari regresi. A salah karena kedua t-stat tidak cukup negatif. C salah karena keduanya (bukan salah satu) punya unit root.
Soal 2 / 4
Hasil Engle–Granger test pada residual regresi (DF t-stat = −3.92, EG 5% CV = −3.34) paling tepat diinterpretasikan sebagai:
A adalah jawaban yang benar. EG test t-stat = −3.92 lebih negatif dari critical value −3.34, sehingga kita tolak H₀: residual punya unit root. Residual stationary → kedua series cointegrated (skenario 5). Regresi valid untuk mengestimasi hubungan jangka panjang (elastisitas 1.12: 1% kenaikan harga batubara → ~1.12% kenaikan harga BUMI). B salah karena −3.92 memang lebih negatif dari −3.34 (kita berhasil tolak). C keliru menyimpulkan arah.
Soal 3 / 4
Berdasarkan hasil ARCH(1) test (koefisien = 0.342, t-stat = 4.18), kesimpulan yang paling tepat adalah:
C adalah jawaban yang benar. t-stat 4.18 >> 1.97 (critical value 5%, dua sisi), sehingga H₀: a₁ = 0 ditolak dengan kuat → ada ARCH(1). Dua konsekuensi: (1) standard error AR(1) model invalid, hypothesis test pada koefisien AR bisa salah → perlu GLS atau metode yang akomodir heteroskedastisitas. (2) Upside: variance error sekarang bisa diforecast pakai σ̂²t+1 = â₀ + â₁ε̂²t. A salah (ARCH jelas ada). B salah (ARCH berdampak pada standard error).
Soal 4 / 4
Menggunakan ARCH(1) model (â₀ = 0.0082, â₁ = 0.342) dan error bulan lalu sebesar +8.0%, forecast variance untuk bulan berikutnya paling mendekati:
B adalah jawaban yang benar. Gunakan formula: σ̂²t+1 = â₀ + â₁ × ε̂²t = 0.0082 + 0.342 × (0.08)² = 0.0082 + 0.342 × 0.0064 = 0.0082 + 0.00219 ≈ 0.01039. Standar deviasi yang diforecast = √0.01039 ≈ 10.2%. A salah karena mengabaikan kontribusi ε̂²t (itu kondisi jika error kemarin = 0). C salah karena menggunakan |ε| bukan ε² (0.0082 + 0.342 × 0.08 = 0.0356). Key takeaway: error besar di periode t menaikkan predicted variance di periode t+1 karena koefisien a₁ positif — inilah volatility clustering.
